PART 3: CROSS-PROCESS DB LOCK
==================================================

Because this project runs multiple bot processes, asyncio.Lock is not enough.

Add a file-based lock for DB writes.

Use a lock file:
casino_bot.db.write.lock

All DB writes must acquire this file lock before BEGIN IMMEDIATE.

Apply to:
- /mine
- /minerrepair
- /send
- /tip
- poker buy-ins/refunds
- blackjack bets/refunds
- bot heartbeat writes if any

If using Python, use filelock if available.
If not, implement a simple lock file with timeout.

Pseudo behavior:
1. acquire file lock with timeout 10s
2. open SQLite connection
3. PRAGMA busy_timeout=15000
4. BEGIN IMMEDIATE
5. write
6. commit
7. close connection
8. release file lock

Never hold the file lock while sending chat/whisper.
