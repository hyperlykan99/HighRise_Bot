Build a full command integrity audit and repair system for my Highrise multi-bot project.

Current problem:
Many slash commands are inconsistent. Some commands work, some are recognized by /commandtest but do not respond, some are in help but not routed, and some cause duplicate replies from Host and the module bot.

Known issues:
- /bal replies from both BankingBot and EmceeBot.
- /mine works from GreatestProspector, but /commandtest /mine does not work.
- /pokerhelp works from ChipSoprano, but /commandtest /pokerhelp does not work.
- /shophelp was fixed.
- Events and AutoGames had WARN issues.
- Host fallback is too aggressive for action/economy/game commands.

Goal:
Create a reliable command integrity system so every command has:
1. A known command entry.
2. A command owner.
3. A route in the main command handler.
4. A valid handler function.
5. Correct bot ownership.
6. No duplicate responders.
7. Safe fallback behavior.
