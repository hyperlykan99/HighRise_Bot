Step 5 — Permissions

Permission levels:

Player:
- Can ask questions
- Can ask what to do next
- Can ask about commands
- Can ask about Luxe, mining, fishing, casino, events
- Can submit bugs/feedback

Staff:
- Everything player can do
- Can ask for moderation help
- Can summarize player reports
- Cannot change economy
- Cannot change odds
- Cannot change prices

Admin:
- Can prepare setting/event changes
- Must confirm before applying
- Cannot wipe data
- Cannot silently change economy/odds/prices

Owner:
- Can prepare protected changes
- Must confirm risky changes
- Can apply confirmed safe actions
- Still cannot wipe player data through AI

Step 6 — Safety rules

Hard rules:
- AI must never wipe/reset/delete player data.
- AI must never grant ChillCoins casually.
- AI must never grant Luxe Tickets casually.
- AI must never change casino odds silently.
- AI must never change prices silently.
- AI must never change economy rewards silently.
- AI must never apply admin/owner changes from one message only.
- AI must always require confirmation for settings changes.
- AI must log every confirmed action.
- AI must deny unauthorized users politely.

Step 7 — Confirmation system

When owner/admin requests a setting change, AI should only prepare it.

Example:

User:
“AceSinatra, set VIP price to 600 tickets.”

AI:
“Prepared change:
Setting: VIP Price
Current: [current value]
New: 600 🎫 Luxe Tickets
Risk: Economy-impacting

Reply CONFIRM VIP PRICE to apply, or CANCEL.”

Only apply the change if:
- Same user replies exact confirmation phrase
- Confirmation has not expired
- User has permission
- Action is allowed by safety rules

Add timeout for pending confirmations.
