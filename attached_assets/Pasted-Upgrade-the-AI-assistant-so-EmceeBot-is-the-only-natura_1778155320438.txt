Main behavior I want:
Users should talk to EmceeBot like this:
- "EmceeBot, can you send testuser 1000 coins?"
- "EmceeBot, send 1000 coins to testuser"
- "EmceeBot, show my balance"
- "EmceeBot, how do I mine?"

Only EmceeBot/host should respond to these AI requests.

Do not make all bots respond to natural language.
Do not trigger AI from random chat.
Do not change bot outfit code.
Do not change bot tokens, room ID, balances, mining rewards, poker state, blackjack state, shop prices, bank settings, event rewards, or player data.

AI trigger rules:
1. AI should activate only when:
   - message mentions "EmceeBot" case-insensitively
   - message starts with "EmceeBot,"
   - message starts with "@EmceeBot"
   - user uses /ask <message>
   - user uses /ai <message>
   - user uses /assistant <message>
   - user has a pending AI action and replies yes/no

2. Do NOT activate AI for general messages like:
   - "show my balance"
   - "how do I mine"
   - "send 100 coins to testuser"
   unless the message mentions EmceeBot or uses /ask.

3. Natural confirmation:
   If the user has a pending AI action, allow:
   yes, y, confirm, do it, proceed, go ahead, sure, ok, okay

   To cancel:
   no, n, cancel, stop, nevermind, never mind

4. If there is no pending action, "yes" or "no" should do nothing dangerous.

Single-bot dispatcher design:
- EmceeBot/host is the only AI assistant voice.
- EmceeBot interprets the request.
- EmceeBot finds the correct command.
- EmceeBot finds the owner bot/module.
- EmceeBot asks confirmation if the action is risky.
- After confirmation, the correct module bot executes the command.
- Module bots should not answer natural AI chat.
