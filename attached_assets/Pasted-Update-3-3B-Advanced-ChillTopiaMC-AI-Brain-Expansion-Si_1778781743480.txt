1. Single Host AI Responder Lock

Create:
- ai_host_lock.py

Purpose:
Prevent every bot from answering AI messages.

Only ChillTopiaMC should respond to AI triggers.

Rules:
- If this running bot is not the configured host AI bot, ignore all AI triggers.
- Only the bot account named ChillTopiaMC should process “ai” messages.
- Other bots may still run their normal commands and systems.
- Other bots must not answer AI assistant messages.
- This prevents duplicate AI replies.

Add config setting:

AI_HOST_BOT_NAME = "ChillTopiaMC"

Add helper function:

def is_ai_host_bot(current_bot_name: str) -> bool:
    return current_bot_name.lower() == AI_HOST_BOT_NAME.lower()

In the message handler:

if is_ai_trigger(message):
    if not is_ai_host_bot(current_bot_name):
        return
    await ai_brain.handle_ai_message(...)

Important:
- Do not disable other bots completely.
- Only disable AI responses on non-host bots.
- Normal commands for other bots should still work if they already exist.
- The AI host lock should happen before intent routing.
- Log ignored AI triggers only in debug mode, not public chat.

Expected:

Only ChillTopiaMC responds.
No other bot should respond.

2. AI Reply Mode Setting

Create:
- ai_reply_mode.py

Goal:
Owner can choose whether AI replies are public or whispered.

Supported modes:

PUBLIC:
- AI replies in public room chat.
- Everyone can see the answer.

WHISPER:
- AI replies privately/whispers to the requesting user.
- Best for personal stats, balances, private help, staff/admin info, and owner info.

SMART:
- AI automatically chooses public or whisper based on content sensitivity.

Default:
AI_REPLY_MODE = "smart"

Owner settings:
- ai set reply mode to public
- ai set reply mode to whisper
- ai set reply mode to smart
- ai reply mode
- ai what is reply mode?

Permission:
Only owner can change global AI reply mode.
Admins may view it, but only owner can change it unless existing permission system allows admin config changes.

Changing reply mode must require confirmation because it affects room behavior.

Example:
Owner:
ai set reply mode to whisper

AI:
Prepared assistant setting change:
Setting: AI Reply Mode
Current: smart
New: whisper
Risk: Medium — changes how AI replies are shown

Reply CONFIRM AI REPLY MODE to apply, or CANCEL.

After confirmation:
AI reply mode updated to whisper.

Reply mode behavior:

PUBLIC mode:
- All AI responses go to public chat unless safety requires whisper.
- Private player data should still be whispered even if global mode is public.
- Staff/admin/owner summaries should still be whispered.

WHISPER mode:
- All AI responses are whispered/private to the requester if whisper is available.
- If whisper is not available in the current bot SDK, fallback to public with private data hidden.
- Do not expose private info publicly if whisper fails.

SMART mode:
AI chooses based on response type.

Public responses:
- General questions
- Public ChillTopia guides
- Luxe Tickets explanation
- ChillCoins explanation
- Mining/fishing guides
- Event public info
- Jokes
- Fun facts
- Basic real-world explanations

Whisper responses:
- Player balances
- Player mission progress
- Player inventory
- Player cooldowns
- Player personalized progress
- Staff reports
- Bug summaries if staff-only
- Moderation guidance
- Admin settings
- Owner analytics
- Economy dashboard
- Confirmation prompts
- Permission denials involving private data
- Any sensitive or private answer

Add helper function:

def choose_ai_reply_channel(reply_mode, response_type, knowledge_level, contains_private_data=False):
    if contains_private_data:
        return "whisper"

    if knowledge_level in ["PLAYER_PRIVATE", "STAFF", "ADMIN", "OWNER"]:
        return "whisper"

    if response_type in [
        "confirmation_preview",
        "owner_summary",
        "staff_summary",
        "moderation_help",
        "player_private_summary",
        "permission_denied_private"
    ]:
        return "whisper"

    if reply_mode == "public":
        return "public"

    if reply_mode == "whisper":
        return "whisper"

    if reply_mode == "smart":
        return "public"

    return "public"

Required safety:
- Never show another player’s private info publicly.
- Never show owner/admin/staff information publicly.
- Never show confirmation prompts publicly if they include sensitive settings.
- Never show private data publicly even if reply mode is public.
- If whisper fails, send a safe public message:
  “I need to whisper that because it contains private info, but whisper is unavailable right now.”
