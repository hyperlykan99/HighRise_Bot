==================================================
INTERFERENCE PREVENTION
==================================================

Very important:
Do not let these aliases break existing commands.

Rules:
1. /p <number> should only mean poker join if the second token is a valid positive number.
   Example:
   /p 1000 = poker join 1000
   /p hello = show "Use /p <buyin>."

2. /r <number> should only mean poker raise if the second token is a valid positive number.
   Example:
   /r 100 = poker raise 100
   /r hello = show "Use /r <amount>."

3. /ps should not override any existing profile/settings command if one exists.
   If /ps already exists, do not use /ps. Use /pstats only.

4. /pp should not override any existing command if one exists.
   If /pp already exists, do not use /pp. Use /pplayers only.

5. /f should only fold if the user is currently in an active poker hand.
   If user is not in poker, say:
   "You are not in poker."

6. /check, /call, /raise, /fold, /allin should only work for poker if:
   - there is an active poker table, and
   - the user is in the active poker hand.
   Otherwise reply:
   "No poker hand active. Use /p <buyin>."

7. Do not use /b, /s, /h, /c as aliases because they may conflict with balance, shop, help, casino, or other commands.

8. Do not override:
   - /help
   - /shop
   - /balance
   - /bank
   - /bj
   - /rbj
   - /send
   - /profile
   - /daily
   - /event
   - /report
   - /rules
   - /gold
   - /vip
   - /quest
