Add these commands:
1. /botoutfits [page]
   Shows all outfits in pages.

2. /botoutfit <mode_id>
   Shows details for one mode outfit.

3. /savebotoutfit <mode_id> [name]
   Saves the current bot outfit into that mode.
   If name is included, update outfit_name too.

4. /dressbot <mode_id>
   Applies the saved outfit data for that mode.
   If outfit data is missing, clearly say:
   “No saved outfit data for this mode. Dress the bot manually, then use /savebotoutfit <mode_id>.”

5. /copyoutfit <username> <mode_id>
   Attempts to copy a user’s outfit and save it to a bot mode.
   Important:
   - First check whether the Highrise SDK supports reading another user’s outfit.
   - If supported, save that user’s outfit into outfit_data_json for the selected mode.
   - If not supported, reply:
     “Copying another user’s outfit is not supported by this SDK. Ask the user to dress the bot manually, then use /savebotoutfit <mode_id>.”
   - Do not crash if the target user is offline or outfit access fails.

6. /wearuseroutfit <username>
   Attempts to make the bot wear the user’s outfit immediately.
   Same SDK rules:
   - If supported, apply it.
   - If not supported, show a friendly unavailable message.
   - Do not crash.

7. /renamebotoutfit <mode_id> <name>
   Renames the saved outfit for that mode.

8. /clearbotoutfit <mode_id>
   Clears saved outfit_data_json for that mode but keeps the mode.
