Notification System Remodel — Manual Subscribe Only, No Auto-Subscribe From Tips

Current issue:
When a player tips a bot, the bot auto-subscribes them to notifications and repeatedly DMs:

✅ Alerts ON. Use notifysettings to choose alerts. Stop: reply unsubscribe.

This is annoying and should stop.

Goal:
Tipping should NEVER auto-subscribe a player.
Players must manually subscribe through direct message only:
!sub
!subscribe

Do NOT modify:
- Poker V2
- blackjack
- jail logic
- Luxe Tickets conversion, except removing notification auto-subscribe from tip flow
- unrelated games

Only modify:
- notification / alert modules
- DM command handling
- tip handler only where it triggers subscription
- help text if needed

==================================================
PART 1 — Remove Tip Auto-Subscribe
==================================================

Search entire project for notification auto-subscribe logic.

Search terms:
autosubscribe
auto_subscribe
subscribe
subscribed
Alerts ON
notifysettings
unsubscribe
enable dm
enabledm
tip
tip_reaction
gold tip

Remove or disable any code path where:
- player tips bot
- bot calls subscribe/enable alerts/enable DM notifications
- bot sends “Alerts ON” automatically
- bot creates notification subscription after tip

Important:
Tip flow should only do tip reward logic:
- award Luxe Tickets
- send tip acknowledgement
- log tip

Tip flow should NOT:
- subscribe user
- enable alerts
- DM “Alerts ON”
- send notifysettings message

==================================================
PART 2 — Manual Subscribe Only By DM
==================================================

A player can only subscribe by sending the bot a direct message:

!sub
!subscribe

When received in DM/private message:

✅ Alerts ON.
You are subscribed to ChillTopiaMC notifications.
Use !unsub to stop.
Use !notifysettings to choose alerts.

Aliases:
!sub
!subscribe

Do not require a tip.
Do not auto-subscribe from room chat.

==================================================
PART 5 — Subscription Database
==================================================

If database.py can be safely modified, add:

CREATE TABLE IF NOT EXISTS notification_subscriptions (
    user_id TEXT PRIMARY KEY,
    username TEXT DEFAULT '',
    subscribed INTEGER DEFAULT 0,
    events INTEGER DEFAULT 1,
    games INTEGER DEFAULT 1,
    tips INTEGER DEFAULT 0,
    promos INTEGER DEFAULT 1,
    source TEXT DEFAULT 'manual',
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);

Rules:
- subscribed = 1 only when user manually uses !sub / !subscribe
- subscribed = 0 when user uses !unsub / unsubscribe / stop
- source must be "manual"
- Never set source = "tip"
- Never create subscribed=1 from a tip event

If DB fails:
- do not crash bot
- print warning
- keep safe default: not subscribed

==================================================
PART 6 — Prevent Repeat “Alerts ON” Spam
==================================================

If player already subscribed and sends !sub again:

You are already subscribed.
Use !notifysettings to edit alerts or !unsub to stop.

Do NOT send repeated “Alerts ON” every time they tip.

If player tips while subscribed:
- no notification subscription message
- only tip reward message

If player tips while unsubscribed:
- no subscription message
- only tip reward message

==================================================
PART 7 — DM vs Room Chat Rules
==================================================

Subscription commands should work best in DM.

If player types !sub in public room chat:
Whisper:
Send me !sub in DM to subscribe to notifications.

Or allow it, but do NOT spam public chat.

Preferred:
- Public !sub replies with whisper only.
- DM !sub confirms subscription.

Unsubscribe can work in both DM and whisper/public, but reply privately if possible.

==================================================
PART 8 — Notification Broadcast Rules
==================================================

Only send notification DMs to users where:
subscribed = 1

Before sending any alert:
check subscription status.

If not subscribed:
do not DM.

If subscribed but category off:
do not DM.

Examples:
Auto event alert:
- send only if subscribed=1 and events=1

Game promo:
- send only if subscribed=1 and games=1 or promos=1

Tip alerts:
- send only if subscribed=1 and tips=1
