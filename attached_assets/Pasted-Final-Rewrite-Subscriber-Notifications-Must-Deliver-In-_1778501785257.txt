Final Rewrite — Subscriber Notifications Must Deliver In-Room Whispers

Current issue:
Players subscribed to notifications are still not receiving bot direct messages, even if they have directly messaged the bot before.

Problem:
Highrise DM/conversation delivery is unreliable or not implemented correctly.
The bot should not depend on conversation DM for players who are currently in the room.

Goal:
Rewrite notification delivery so subscribed players currently in the room always receive whispers.
Out-of-room delivery should try SDK bulk/user DM if available, then conversation DM if available, otherwise log unavailable.

Do not change bot tokens.
Do not change room ID.
Do not reset subscriptions.
Do not wipe notification preferences.
Do not wipe notification logs.
Do not break /notif, /notifon, /notifoff, /notifall, /subnotify, /subnotifyinvite, /subnotifystatus, or /testnotify.

==================================================
PART 1 — DELIVERY PRIORITY
==================================================

New delivery priority:

1. If eligible subscribed player is currently in room:
   - send whisper
   - status = sent_whisper_in_room
   - delivery_method = whisper
   - do not require conversation_id
   - do not attempt DM first

2. If player is not currently in room:
   - try Highrise send_message_bulk to user_id if SDK supports it
   - if successful:
     status = sent_bulk_dm
     delivery_method = bulk_dm

3. If bulk DM unavailable or fails:
   - try conversation DM using saved conversation_id
   - if successful:
     status = sent_conversation_dm
     delivery_method = conversation_dm

4. If no delivery works:
   - status = no_delivery_route
   - do not count as sent

Never mark logged-only as sent.

==================================================
PART 2 — ROOM USER DETECTION
==================================================

Before sending /subnotify, refresh current room users.

Use the best available method:
- highrise.get_room_users()
- existing room user tracker
- on_user_join/on_user_leave tracker
- on_chat recent seen tracker

If SDK supports get_room_users:
Use it as source of truth.

If not:
Use current_room_users memory plus recent_seen within 10 minutes.

If user is found in room:
send whisper.

==================================================
PART 3 — SEND WHISPER TO IN-ROOM PLAYERS
==================================================

For each eligible player in room:

await bot.highrise.send_whisper(user_id, message)

Log:
status = sent_whisper_in_room
delivery_method = whisper
sender_bot_name = selected category bot

This must work even if:
- player has no DM conversation
- conversation_id is missing
- DM SDK is unavailable

==================================================
PART 4 — OUT-OF-ROOM DELIVERY
==================================================

For eligible players not in room:

First try bulk/user DM if SDK supports it.

Check for method availability safely:
- hasattr(bot.highrise, "send_message_bulk")

If available:
- send to user_id
- use message_type text or required SDK format
- if invite command, use invite type + room_id/world_id if available

If not available:
- skip to conversation DM attempt

Then try conversation DM:
- use saved conversation_id only
- call send_message(conversation_id, message, type="text")
- if conversation not found, log no_conversation

Statuses:
- sent_bulk_dm
- sent_conversation_dm
- no_conversation
- unsupported_bulk_dm
- unsupported_conversation_dm
- failed
