Fix Jail Setup Commands — Real Routing Not Just Commandtest

Current issue:
!commandtest !setjailspot shows:
owner=KeanuShield
host_handles=false
route=YES
handler=YES

But typing:
!setjailspot

does not save the jail spot and does not reply.

Root cause:
Commandtest only checks audit metadata/registry, not actual execution.
multi_bot.py blocks host from handling security-owned commands because security is in _HARD_OWNER_MODES.

Goal:
KeanuShield/security should own the jail system, but ChillTopiaMC/host must be allowed to run jail setup commands as owner/admin override.

Security bot:
KeanuShield
BOT_MODE=security

Keep owner:
Jail commands owner should remain security/KeanuShield.

Do NOT change owner to ChillTopiaMC.

Add a special whitelist in modules/multi_bot.py:

_JAIL_SETUP_HOST_OVERRIDE_CMDS = frozenset({
    "setjailspot",
    "setjailguardspot",
    "setsecurityidle",
    "setjailreleasespot",
    "jaildebug",
    "jailadmin",
})

Then in should_this_bot_handle(cmd), add this BEFORE the hard-owner block:

if mode in ("host", "all") and owner_mode == "security" and cmd in _JAIL_SETUP_HOST_OVERRIDE_CMDS:
    return True

Important:
This must happen before:

if mode in ("host", "eventhost") and owner_mode in _HARD_OWNER_MODES:
    return False

Runtime jail commands should NOT use host override:
- jail
- bail
- unjail
- jailrelease
- jailactive
- jailstatus
- jailtime
- jailconfirm
- jailcancel

Those should stay owned by KeanuShield/security.
