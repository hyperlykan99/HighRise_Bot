Add global player display formatting with badges and titles.

Goal:
Whenever any bot mentions, calls, announces, or displays a player’s name, it should include that player’s equipped badge and/or title.

Examples:
Old:
@Marion won 5,000 coins.

New:
💎 [High Roller] @Marion won 5,000 coins.

Old:
@Claire joined poker.

New:
🌟 [VIP] @Claire joined poker.

Do not change bot tokens.
Do not change room ID.
Do not reset balances.
Do not change poker, blackjack, mining, bank, shop, events, outfits, bot spawns, or game rules.

Use existing badge/title system.
Do not create a new badge/title system unless missing.
Do not expose private data.

==================================================
PART 1 — GLOBAL DISPLAY HELPER
==================================================

Create or update a shared helper function, for example:

format_player_display(username, user_id=None, include_badge=True, include_title=True)

or:

get_player_display_name(username)

It should return a short formatted player display string.

Recommended format:
<badge_emoji> [<title>] @username

Examples:
💎 [High Roller] @Marion
🌟 [VIP] @Claire
⛏️ [Master Miner] @testuser
@newplayer

Rules:
1. If player has equipped badge, show badge emoji/icon.
2. If player has equipped title, show title in brackets.
3. If no badge/title, show normal @username.
4. Keep it short.
5. Do not show raw badge IDs unless no display name exists.
6. Do not break if user has no profile yet.
7. Do not crash if badge/title data is missing.
8. Cache display lookup if needed to avoid database spam.

==================================================
PART 3 — MESSAGE SAFETY
==================================================

Do not make messages too long.

If display is too long, fallback to:
<badge> @username

If still too long, fallback to:
@username

Do not exceed Highrise message length limits.

Do not use badge/title formatting in commands where username must be parsed as an argument.
Only use it for display output, not command execution.

Example:
Internal command should still use:
send Marion 100

Displayed message can use:
💎 [High Roller] @Marion sent 100 coins.

==================================================
PART 5 — SETTINGS / TOGGLES
==================================================

Add optional staff setting:

/displaybadges on
/displaybadges off
/displaytitles on
/displaytitles off
/displayformat

Behavior:
/displayformat shows current format:
Badge: ON
Title: ON
Format: <badge> [title] @username

Owner: Host or Shop, whichever manages titles/badges.

If you think toggles are unnecessary, still add at least /displayformat for debugging.
