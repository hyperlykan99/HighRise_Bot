Implementation Part 1 — Fix real !jail runtime dispatch

In main.py, add jail runtime routing outside the STAFF_CMDS block.

Place this after multi-bot gate and before maintenance/mute gates:

if cmd in {
    "jail", "bail", "jailstatus", "jailtime", "jailhelp",
    "jailactive", "unjail", "jailrelease",
    "jailconfirm", "jailcancel"
}:
    print(f"[JAIL ROUTE HIT] runtime cmd={cmd} user={user.username} bot_mode={BOT_MODE}")

    if cmd == "jail":
        await handle_jail(self, user, args)
    elif cmd == "bail":
        await handle_bail(self, user, args)
    elif cmd in ("jailstatus", "jailtime"):
        await handle_jailstatus(self, user, args)
    elif cmd == "jailhelp":
        await handle_jailhelp(self, user, args)
    elif cmd == "jailactive":
        await handle_jailactive(self, user, args)
    elif cmd in ("unjail", "jailrelease"):
        await handle_unjail(self, user, args)
    elif cmd == "jailconfirm":
        await handle_jail_confirm(self, user)
    elif cmd == "jailcancel":
        await handle_jail_cancel(self, user)
    return

Important:
Do not require !jail to be in STAFF_CMDS.
Players must be able to use !jail if jail system is enabled and they pay Luxe Tickets.

Implementation Part 4 — Add jail reason

Support:
!jail @user [minutes] [reason...]

Examples:
!jail @AshQuinn_ 1 testing jail
!jail @AshQuinn_ 3 being too loud
!jail @AshQuinn_

Parsing:
- args[1] = target username
- args[2] = minutes if numeric, otherwise default minutes
- remaining args after minutes = reason
- if no reason: reason = "No reason given"

Examples:

target = AshQuinn_
minutes = 1
reason = testing

!jail @AshQuinn_ being silly
