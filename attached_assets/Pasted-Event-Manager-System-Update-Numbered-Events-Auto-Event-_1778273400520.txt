Event Manager System Update — Numbered Events, Auto Event Pool, Scheduler Status, and Mining Events

Focus only on the Event Manager / EventBot system.

Do not change bot tokens.
Do not change room ID.
Do not reset balances.
Do not wipe player data.
Do not break mining, poker, blackjack, bank, shop, AI, outfits, bot spawns, or existing commands.

Goal:
Create a clean Event Manager system where EventBot/EventHost controls all events, auto-events, event scheduling, event pools, event history, event cooldowns, and event status.

MiningBot should NOT control event timers.
MiningBot should only read active mining event effects and apply boosts.

==================================================
PART 1 — EVENT MANAGER CORE RULES
==================================================

EventBot/EventHost should control:

- normal room events
- mining events
- auto events
- event timers
- active event effects
- event schedules
- event pool
- event queue
- event history
- event cooldowns
- event weights
- event persistence after restart
- scheduler heartbeat
- duplicate scheduler protection

Events must persist after restart.

If the bot crashes/restarts:
1. Active events should reload from database.
2. If an event is still active, continue it with remaining time.
3. If an event expired during downtime, mark it expired.
4. Do not restart expired events.
5. Do not duplicate active events.

Only EventBot/EventHost should run the event scheduler loop.
