User:

Good behavior:
Run /profile testuser or /whois testuser using the existing profile/whois handler.

User:

Good behavior:
Run /bal through the banker/bank handler or delegate to BankingBot.

User:
Emcee, show shop

Good behavior:
Run /shop or call the shop handler.

Only suggest commands if the system cannot safely execute it.

PART 2 — Add smart “cannot do that yet” responses

If a requested feature does not exist in the command registry, route table, or handler system, do not hallucinate.

Reply with one of these:

1. If no command exists:
“I can’t do that yet. I don’t have a working command for that feature.”

2. If command exists but handler is missing:
“I found that command, but it is not fully coded yet.”

3. If owner bot is offline:
“I can do that, but the required bot is offline right now.”

4. If SDK does not support it:
“I can’t do that yet because the Highrise SDK does not support that action here.”

5. If permission is missing:
“You don’t have permission to do that.”

PART 5 — Delegated task system

If EmceeBot needs another bot to execute a task, use the existing delegation system or add one.

Suggested table:
ai_delegated_tasks:
- id
- user_id
- username
- original_text
- command_text
- owner_mode
- owner_bot_username
- target_bot_username
- human_readable_action
- risk_level
- status
- created_at
- expires_at
- completed_at
- error

Flow:
1. EmceeBot interprets request.
2. EmceeBot checks permissions.
3. EmceeBot asks confirmation if needed.
4. User says yes.
5. EmceeBot creates delegated task.
6. Target bot checks for tasks for itself every 1–2 seconds.
7. Target bot executes using existing handler.
8. Mark task completed or failed.
9. EmceeBot or target bot reports result.
