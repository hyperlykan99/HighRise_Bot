Rewrite Subscriber Notification System From Scratch

Current issue:
Players are subscribed and opted into categories, but they are still not receiving direct messages from subscriber notifications.

Goal:
Replace the current notification system with a clean, reliable system that:
1. Tracks notification preferences by category.
2. Tracks whether a player can receive DM/inbox messages.
3. Falls back to whisper if the player is currently in the room.
4. Accurately reports sent, skipped, no conversation, unsupported SDK, and failed.
5. Never says a message was sent if it was only logged.
6. Does not spam players.
7. Does not break subscriptions, rewards, events, or existing player data.

Do not change bot tokens.
Do not change room ID.
Do not reset subscriptions.
Do not wipe notification preferences unless migration is required.
Do not wipe player data.
Do not break /notif, /notifon, /notifoff, /notifall, /subnotify, /subnotifyinvite, or /subnotifystatus.
Do not expose secrets, tokens, env vars, or room ID.

PART 9 — DELIVERY RULES
==================================================

When /subnotify runs:

For each player:

1. Check if player is subscribed.
   If not subscribed:
   status = skipped_not_subscribed

2. Check global notification setting.
   If global_enabled = 0:
   status = skipped_global_off

3. Check category setting.
   If category enabled = 0:
   status = skipped_category_off

4. If eligible, delivery order:

A. If DM conversation_id exists and SDK supports send_message:
   - send DM/inbox message
   - status = sent_dm
   - delivery_method = dm

B. Else if player is currently in the room:
   - send whisper
   - status = sent_whisper_in_room
   - delivery_method = whisper

C. Else:
   - status = no_conversation
   - delivery_method = none

5. If SDK does not support DM sending:
   - status = unsupported_sdk for DM attempt
   - still try whisper if player is in room

6. If any send throws an exception:
   - status = failed
   - save safe error
   - print full traceback to console

Important:
Do not count no_conversation as sent.
Do not count unsupported_sdk as sent.
Do not count failed as sent.
