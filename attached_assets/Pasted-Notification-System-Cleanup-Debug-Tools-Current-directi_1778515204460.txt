PART 2 — REQUIRED DELIVERY BEHAVIOR
==================================================

For each eligible subscribed player:

If player is:
- subscribed
- global notifications ON
- category ON
- currently in room

Then:
- EmceeBot whispers them immediately.
- status = sent_whisper_in_room
- delivery_method = whisper

Do not require DM conversation for players currently in room.
Do not attempt DM before whisper for in-room players.

If player is:
- subscribed
- global notifications ON
- category ON
- NOT currently in room

Then:
- try DM only if SDK supports it.
- if DM works: status = sent_dm or sent_conversation_dm / sent_bulk_dm depending current code.
- if no route: status = no_delivery_route.

Do not mark no-route, failed, skipped, or unsupported as sent.

PART 6 — /subnotify
==================================================

Command:

/subnotify <category> <message>

Example:

/subnotify events First Hunt starts in 5 minutes!

Behavior:
- Validate category.
- Load subscribed players.
- Check global notification setting.
- Check category setting.
- Refresh room users.
- Whisper in-room eligible players from EmceeBot.
- Try DM for out-of-room eligible players only if supported.
- Log all recipients with accurate statuses.
- Reply to sender with accurate summary.

Summary example:

EmceeBot:
🔔 Subscriber Notification Complete
Category: Events
Sender: EmceeBot
Whisper Sent In Room: 9
DM Sent Out of Room: 1
No Delivery Route: 2
Skipped Opt-Out: 3
Failed: 1

If no messages delivered:

EmceeBot:
🔔 Subscriber Notification
No messages were delivered.
Reason: No eligible subscribed players currently in room and out-of-room DM is unavailable.

Important:
Do not count no_delivery_route as sent.
Do not count failed as sent.
Do not count skipped as sent.
