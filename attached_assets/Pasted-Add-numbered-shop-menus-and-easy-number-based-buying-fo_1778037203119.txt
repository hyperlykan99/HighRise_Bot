Add numbered shop menus and easy number-based buying for all shops.

GOAL:
Make every shop easier for players.
Instead of typing long item IDs, players should see numbered items and buy by number.

Example:
Bot shows:
/shop badges
1. ⭐ Star — 500c
2. 🔥 Fire — 2,500c
3. 💎 Diamond — 10,000c

Player types:
/buy 2

Bot buys item #2 from the last shop page they viewed.

IMPORTANT:
- Do not remove existing shop commands.
- Existing commands like /buy badge diamond, /buy title elite, /buyvip, /eventshop buy <id> should still work.
- Add numbered buying as an easier option.
- Use SQLite.
- Use number formatter with commas and short format.
- Keep every Highrise message under 250 characters.
- Split long shop pages.
- Prevent crashes from bad inputs.
- Add explicit command routing before unknown-command fallback.

==================================================
PART 1: NUMBERED SHOP SESSION
==================================================

Create table shop_view_sessions:

shop_view_sessions:
- username TEXT PRIMARY KEY
- shop_type TEXT
  Values: badges, titles, vip, event, general, market_badges, market_titles
- page INTEGER DEFAULT 1
- items_json TEXT
- viewed_at TEXT

When a player views any shop page, save the exact items shown to them in shop_view_sessions.

Important:
- /buy <number> should buy from the player’s most recently viewed shop page.
- Session should expire after 10 minutes.
- If expired:
"Shop selection expired. Open the shop again."

==================================================
PART 3: EASY BUY COMMANDS
==================================================

Add/fix:
- /buy <number>
- /buyitem <number>
- /purchase <number>

Behavior:
- Looks up the user’s latest shop_view_sessions.
- Finds item by number on that saved page.
- Executes correct purchase action based on shop_type and item data.

Examples:
After /shop badges:
- /buy 2 buys badge #2.

After /shop titles:
- /buy 1 buys title #1.

After /vipshop:
- /buy 1 buys VIP.

After /eventshop:
- /buy 2 buys event shop item #2 using event coins.

Messages:
"✅ Bought 🔥 Fire for 2,500c / 2.5K."
"✅ Bought title [Elite] for 100,000c / 100K."
"✅ VIP purchased."
"✅ Bought Crown Badge for 500 event coins."

If number invalid:
"Invalid number. Open the shop again."

If no recent shop:
"Open a shop first. Example: /shop badges"

If item already owned:
"You already own this."

If not enough coins:
"Not enough coins."

If not enough event coins:
"Not enough event coins."

PART 6: CONFIRMATION FOR EXPENSIVE ITEMS
==================================================

Add confirmation for expensive purchases.

Default threshold:
- shop_confirm_threshold = 100000 coins

If item price >= threshold:
- Do not buy immediately.
- Create pending purchase with code.
- Message:
"Confirm 250K purchase: /confirmbuy ABC123"

Add table pending_shop_purchases:
- code TEXT PRIMARY KEY
- username TEXT
- shop_type TEXT
- item_id TEXT
- item_name TEXT
- price INTEGER
- currency TEXT
- created_at TEXT
- expires_at TEXT

Commands:
- /confirmbuy <code>
- /cancelbuy <code>

If confirmed:
- Re-check balance and ownership.
- Complete purchase.
- Message:
"✅ Purchase complete."

If expired:
"Purchase expired."

Event coin purchases may also use confirmation if expensive:
- event_confirm_threshold = 500 event coins

Owner/admin settings:
- /setshopconfirm <amount>
- /seteventconfirm <amount>
