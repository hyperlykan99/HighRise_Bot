Fix duplicate module startup/restore messages across all bots.

Problem:
All bots are saying:
"♻️ Poker table restored. 1 seated. Waiting for players."

Bots affected:
BankingBot, KeanuShield, ChipSoprano, DJ_DUDU, GreatestProspector, EmceeBot, AceSinatra, TestDJBot.

Goal:
Only the bot that owns a module should run that module's startup recovery and send that module's restore messages.

Example:
- Poker restore messages: only Poker Bot.
- BJ/RBJ restore messages: only Blackjack Bot.
- Mining restore/event messages: only Miner Bot.
- Bank/economy restore messages: only Banker Bot.
- Shop restore messages: only Shop Bot.
- Reports/moderation restore messages: only Security Bot.
- Emote/DJ restore messages: only DJ Bot.
- Event/autogame restore messages: only Event Bot.
- General welcome/help startup messages: only Host Bot.

IMPORTANT:
- Do not remove recovery features.
- Do not stop bots from running.
- Keep shared SQLite database.
- Prevent duplicate restore announcements forever.
- Keep messages under 250 characters.
- If a bot does not own the module, it must not run that module’s recovery loop or speak its restore message.
- Console logs are okay. Room messages must be owner-only.

==================================================
PART 1: MODULE OWNER GUARD
==================================================

Create/fix helper:

should_this_bot_run_module(module_name, bot_mode, bot_id)

Rules:
1. poker module runs only on BOT_MODE=poker.
2. blackjack and rbj modules run only on BOT_MODE=blackjack.
3. mining module runs only on BOT_MODE=miner.
4. bank/economy module runs only on BOT_MODE=banker.
5. shop/badges/titles/vip module runs only on BOT_MODE=shopkeeper.
6. security/reports/moderation module runs only on BOT_MODE=security.
7. dj/emotes/social module runs only on BOT_MODE=dj.
8. events/autogames/announcements module runs only on BOT_MODE=eventhost.
9. host/help/tutorial/welcome/profile module runs only on BOT_MODE=host.
10. BOT_MODE=all can run all modules only if no split bot for that module is active.
11. If split bot for module is active, BOT_MODE=all/host/main must not run that module.

Use bot_instances heartbeat to detect active split bots.

==================================================
PART 2: WRAP ALL STARTUP RESTORE TASKS
==================================================

Find all startup recovery/restore calls and wrap them.

Examples:

Poker:
Before:
restore_poker_table()
announce_poker_restore()

After:
if should_this_bot_run_module("poker", BOT_MODE, BOT_ID):
    restore_poker_table()
    announce_poker_restore()
else:
    console_log("[SKIP] poker restore skipped on " + BOT_MODE)

BJ/RBJ:
if should_this_bot_run_module("blackjack", BOT_MODE, BOT_ID):
    restore_bj()
    restore_rbj()

Mining:
if should_this_bot_run_module("mining", BOT_MODE, BOT_ID):
    restore_mining_events()

Bank:
if should_this_bot_run_module("bank", BOT_MODE, BOT_ID):
    restore_bank_jobs()

Event:
if should_this_bot_run_module("events", BOT_MODE, BOT_ID):
    restore_events()
    start_autogames_if_enabled()

Room/Host:
if should_this_bot_run_module("host", BOT_MODE, BOT_ID):
    restore_welcome_intervals()

==================================================
PART 3: OWNER-ONLY ROOM ANNOUNCEMENTS
==================================================

Create helper:

send_module_room_message(module_name, message)

Before sending, it checks:
- current bot owns module
- module announcement enabled
- not within duplicate cooldown
- message not already sent by another bot

If bot does not own module:
- do not send
- console log only

This applies to:
- poker table restored
- BJ/RBJ table restored
- mining event restored
- autogames started
- welcome interval restored
- shop restored
- bot startup online messages

==================================================
PART 4: RESTORE MESSAGE DEDUPE LOCK
==================================================

Create table if missing:

module_announcement_locks:
- module TEXT PRIMARY KEY
- message_key TEXT
- bot_id TEXT
- sent_at TEXT
- expires_at TEXT

For restore messages:
- module = poker
- message_key = poker_table_restored

Before sending:
1. Try to acquire lock.
2. If lock exists and not expired, skip.
3. If acquired, send message.
4. Lock expires after 5 minutes.

This prevents accidental duplicate messages even if two bots mistakenly reach the same code.

Each task has:
- task_name
- module

- interval_seconds
- enabled

Examples:
