Separate the bot into a Multi-Bot System with shared database and bot-specific roles.

GOAL:
I want to separate my bot features into different bots/personas.
Each bot should handle specific modules, but all bots should share the same database, economy, roles, player profiles, coins, XP, inventory, and logs.

Example bots:
- Host Bot
- Banker Bot
- Dealer Bot
- Miner Bot
- Shop Bot
- Security Bot
- DJ Bot
- Event Bot

IMPORTANT:
- Do not remove existing features.
- Keep all current commands working if only one bot is running.
- Use SQLite shared database.
- Add bot_id and bot_mode support.
- Each bot should only respond to commands assigned to its module unless fallback is enabled.
- Prevent multiple bots from replying to the same command.
- Keep all Highrise messages under 250 characters.
- Add explicit routing before unknown-command fallback.
- Prevent silent failures.
- If only the main bot is running, it should still handle everything.

==================================================
PART 4: PREVENT MULTIPLE BOTS RESPONDING
==================================================

Before responding to a command, each bot must check:

should_this_bot_handle(command, BOT_MODE)

Rules:
1. If BOT_MODE = all, handle everything.
2. If command belongs to this bot mode, handle it.
3. If command owner bot is offline and fallback_allowed = true, main/host bot may handle it.
4. If command belongs to another active bot, ignore silently.
5. Unknown command should only be answered by Host Bot or BOT_MODE all.

This prevents:
- Dealer Bot and Host Bot both replying to /p
- Banker Bot and Host Bot both replying to /bal
- All bots saying unknown command

Unknown command handling:
- Only Host Bot or all-mode bot replies:
"Unknown command. Type /help."

Other bots should ignore unknown commands.

==================================================
PART 13: SAFE FALLBACK BEHAVIOR
==================================================

If multiple bots are running:
- Only owner bot for command responds.

If owner bot is offline:
- If fallback ON, Host/Main responds.
- If fallback OFF, Host says:
"That module bot is offline."

If only one bot is running:
- Main all-mode handles everything.

This makes development easy and production clean.
