Redo and upgrade the Room Utility Core Module with Bot Outfit / Bot Mode system.

GOAL:
Add all basic room bot features plus a bot dress-up system.

Room utility features:
- teleport commands
- spawn system
- roles and permissions
- titles integration
- moderation: mute, ban, kick, warnings
- player lists
- emotes
- force emotes for staff
- looping emotes
- sync dance from host
- heart reactions
- social actions: hug, kiss, slap, punch
- bot follow user
- room alerts
- welcome messages
- recurring interval messages
- safe repeat messages
- staff utilities

New bot outfit features:
- Bot can have different modes/personas.
- Each bot mode has a prefix, title, description, outfit name, and optional role.
- Staff can manually pick the bot’s look/mode.
- Later, multiple bots can use different outfits/modes like Miner, Banker, DJ, Casino Dealer, Host, Security, etc.

IMPORTANT:
- Do not remove existing features.
- Keep coins, bank, shop, badges, titles, VIP, casino, BJ, RBJ, poker, mining, profiles, reports, notifications, and roles.
- Use SQLite.
- Keep all Highrise messages under 250 characters.
- Split long outputs into pages.
- Add explicit command routing before unknown-command fallback.
- Prevent crashes from bad inputs.
- Every command must reply with success, usage, permission error, unsupported API, or clear error.
- If Highrise API does not support an action, reply:
"This action is not supported by the current API."
- Do not fake success for outfit changing, mic, kick, ban, boost, teleport, or emote control.

==================================================
PART 1: ROLE PERMISSIONS
==================================================

Use existing role hierarchy:

Owner
Admin
Manager
Moderator
VIP
Player

Owner:
- Full control.
- Can use all room utility commands.
- Can create/edit/delete bot outfits and bot modes.
- Can assign bot mode to any bot.

Admin:
- Can use moderation, teleport, alerts, emotes, welcome, intervals.
- Can switch bot mode/outfit if allowed.
- Cannot remove owners.

Manager:
- Can use room utility, teleport, alerts, emotes, welcome, events.
- Can switch bot mode if allowed.
- Cannot ban owners/admins.

Moderator:
- Can warn, mute, kick if supported, view reports, use staff alerts.
- Cannot edit bot outfits.

VIP:
- Fun social commands and special emotes only.

Player:
- Public social commands, help, lists, profile, game commands.
