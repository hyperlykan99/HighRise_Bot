Upgrade Poker into a robust, fault-safe poker system.

GOAL:
Make Poker stable, realistic, easy to manage, and hard to break.

Main fixes/features:
- No stuck hands.
- No missing card whispers.
- No unresolved pots.
- Players leaving room auto-fold and leave table.
- AFK players are removed from table after missed actions.
- Table credits/stacks are safely refunded.
- Better help commands with short descriptions.
- Better debug commands with short descriptions.
- Add poker speed modes: normal, long, fast, custom.
- Add buy-in modes: low, normal, high roller, custom.
- Add owner-only AI poker simulation/debug mode to stress-test poker.
- Owner can view all AI/debug cards only in debug mode.

IMPORTANT:
- Do not remove existing poker features.
- Keep Poker Bot ownership.
- Keep multi-bot routing.
- Keep shared SQLite database.
- Keep player balances safe.
- Keep all messages under 250 characters.
- No silent failures.
- Every command must reply with success, usage, permission error, or clear error.
- Never leak real player private cards.
- Owner all-card view is allowed only in AI/debug simulation mode or explicit owner debug command.

==================================================
PART 1: PLAYER LEAVES ROOM AUTO-FOLD
==================================================

When a player leaves the Highrise room:

If player is seated at poker table:
1. If player is in active hand:
   - Auto-fold them.
   - Mark fold reason = "left_room".
   - Remove them from current hand.
   - If only one active player remains, resolve pot immediately.
2. Refund their remaining table stack to main wallet.
3. Remove them from the table seat.
4. Clear pending turn/action for them.
5. Announce:
"🚪 @user left. Auto-folded and stack refunded."

If player is seated but not in active hand:
- Refund remaining table stack.
- Remove from table.
- Message:
"🚪 @user left poker table. Stack refunded."

Rules:
- Do not refund active bets already committed to pot.
- Refund only remaining table stack.
- Use SQLite transaction.
- Log action in poker_logs.
- If room leave event is unavailable, detect using periodic room presence check.

Create/update:
poker_player_presence:
- username_key TEXT PRIMARY KEY
- display_name TEXT
- in_room INTEGER DEFAULT 1
- last_seen_at TEXT
- last_checked_at TEXT

==================================================
PART 2: AFK AUTO-FOLD AND REMOVE
==================================================

If a player fails to act before action timer expires:

First missed action:
- Auto-check if possible.
- If cannot check, auto-fold.
- Increment missed_actions.

Second missed action:
- Auto-fold.
- Mark player as sitting out.
- Message:
"⏳ @user timed out and is sitting out."

Third missed action or setting threshold:
- Remove player from table.
- Refund remaining table stack.
- Message:
"💤 @user removed for AFK. Stack refunded."

Settings:
- poker_afk_enabled = true
- poker_afk_missed_before_sitout = 2
- poker_afk_missed_before_remove = 3
- poker_afk_refund_stack = true

Commands:
- /poker afk on
- /poker afk off
- /setpokerafkremove <misses>
- /setpokerafksitout <misses>

Messages:
"✅ Poker AFK removal ON."
"✅ AFK remove threshold set to 3."

==================================================
PART 3: SAFE REFUND RULES
==================================================

When leaving/AFK/removing player:

Refund:
- player’s remaining table stack

Do not refund:
- chips already bet in current pot
- blinds already posted
- antes already posted

If hand is not active:
- refund full table stack.

If active hand:
- auto-fold first
- refund remaining stack
- keep committed pot in pot

If player is all-in:
- do not remove until hand resolves unless they left room.
- If they left room, keep their all-in hand eligible only if poker rules allow disconnected all-in.
- Add setting:
  poker_left_allin_policy = fold
  allowed values: fold, keep_allin

Default:
- fold

Command:
- /poker leftallin fold
- /poker leftallin keep

PART 5: AUTO-RECOVERY GUARDS
==================================================

Before every action:
- verify table is valid.
- verify player is in hand.
- verify it is their turn unless command is info.
- verify pot >= 0.
- verify stacks >= 0.
- verify cards exist.

If invalid:
- reject action with clear message.
- do not continue corrupt hand.

Examples:
"Poker hand is stuck. Staff: /poker recoverystatus."
"Not your turn."
"You are not in this hand."
"No active poker hand."
