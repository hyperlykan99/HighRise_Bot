UPDATE 3.1I — TIMER-BASED AUTO MINING/FISHING + LUCK STACK REDESIGN

Goal:
Redesign auto-mining and auto-fishing to work like a Prospecting-style system.

Main changes:
1. Remove set catch/count from auto-mining and auto-fishing.
2. Auto-mining and auto-fishing should depend on timer/duration.
3. Duration, speed, and luck depend on VIP, admin settings, tools/rods, potions, boosts, and events.
4. Players can see their full luck stack.
5. Mining and fishing should have clear stack breakdowns.
6. Auto completion summaries should show time, attempts, rewards, best find, rare finds, new discoveries, and luck used.
7. Keep collection book integration.
8. Do not re-add reward caps.

Do not change tokens.
Do not change room ID.
Do not wipe database.
Do not reset balances, VIP, badges, casino, blackjack, poker, leaderboards, collection book, mining inventory, fishing inventory, tools, rods, or player data.
Do not change mining/fishing item tables unless needed safely.
Do not add reward caps.
Do not add cap text.
Do not break !collection, !orebook, !fishbook, !lastminesummary, or !lastfishsummary.

==================================================
PART 1 — REMOVE SET CATCH / SET COUNT AUTO SYSTEM
==================================================

Remove player-selected catch/count logic from:

!automine [count]
!autofish [count]

Players should no longer decide how many mines/casts they get.

New behavior:

!automine
Starts timer-based auto-mining.

!autofish
Starts timer-based auto-fishing.

If player types:

!automine 50
!autofish 50

Reply:

⚠️ Auto sessions are timer-based now.
Use !automine or !autofish.

Rules:
- Do not let players choose count.
- Do not use old count-based auto loops.
- Keep stop/cancel command if already exists.
- Do not break manual !mine and !fish.

==================================================
PART 2 — TIMER-BASED AUTO SYSTEM
==================================================

Auto session should calculate:

auto_duration
action_interval
total_attempts

Formula:

total_attempts = floor(auto_duration_seconds / action_interval_seconds)

Example:
Auto Time: 15 minutes
Speed: 10s/action
Attempts: 90

Rules:
- Duration comes from base settings + VIP + tool/rod + potion + boost + event.
- Action speed comes from base interval reduced by tool/rod + VIP + potion + boost + event.
- Enforce a minimum interval to prevent spam.
- Do not allow zero-second action speed.
- Do not run infinite loops.
