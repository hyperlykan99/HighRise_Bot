Update 3.0M Hotfix — Low-Interference Assistant Mode With No Cooldown + Whisper Replies

Current concern:
If players are talking normally, ChillTopiaMC might keep answering and become annoying.

Goal:
Make ChillTopiaMC helpful but quiet.

Important:
Do NOT use cooldowns.
Instead, use strict intent rules so the assistant only replies when the message clearly needs help.

Also:
If a player whispers/DMs the bot, the bot must reply in whisper/DM.
If a player talks in public chat, the bot may reply publicly only when intent is clear.

Do not change command routing.
Do not let AI execute commands.
Do not change economy, VIP, mining, fishing, casino, admin, tips, notifications, teleports, roles, or spawns.
Do not add advanced AI actions.

==================================================
PART 1 — REMOVE ASSISTANT COOLDOWNS
==================================================

Remove or disable assistant cooldown blocking for:
- greeting replies
- room guide replies
- mining help
- fishing help
- VIP help
- teleport help
- party help
- shop help
- notification help

Reason:
If a player asks different clear questions back-to-back, the bot should answer each one.

Example:
how do i mine
how do i fish
how do i get vip

Expected:
All three should reply.

Do not block different help questions.

==================================================
PART 2 — REPLY LOCATION RULE
==================================================

Assistant replies must match the incoming message type.

If the user whispers/DMs the bot:
- reply by whisper/DM to that user
- do not reply publicly

If the user talks in public chat:
- reply publicly only if intent is clear
- do not whisper unless the response is owner/admin/private

If the user asks owner/admin/private info publicly:
- if owner, whisper the response
- if not owner, reply:
🔒 Owner-only tool.

Expected:
Player whispers:
“How do I fish?”

Bot whispers back:
🎣 Type !fish to fish once. Use !autofish for auto fishing, !fishinv for inventory, and !fishchances for odds.

Expected:
Player public chat:
“How do I fish?”

Bot public reply:
🎣 Type !fish to fish once. Use !autofish for auto fishing, !fishinv for inventory, and !fishchances for odds.

==================================================

==================================================

Assistant may reply only if one of these is true:

1. Message mentions bot name:
ChillTopiaMC
chilltopia

2. Message is a DM/whisper to the bot.

3. Message starts with a clear help phrase:
how do i
how to
what can i do
what games

where can i
help

4. Message is a direct room topic question:
how do i mine

how do i teleport
what is party mode
what is vip
what games are here
how do i subscribe
how do i buy
how do i play blackjack
how do i play poker

5. Message is an unknown !command.

If none of these are true:
Do not reply.

Important:
Whisper/DM messages are allowed to be answered more often because they are private, but still do not execute restricted actions.

==================================================

Do not add:
- global assistant cooldown
- per-user assistant cooldown

- max replies per 5 minutes

Instead, rely only on strict intent matching.

If the message is clear help intent, reply.
If not clear, ignore.

==================================================
PART 6 — COMMAND ROUTER SAFETY
==================================================

All messages starting with ! must go through command router first.

Rules:
- Known !commands route normally.
- Unknown !commands use unknown command handler.
- Do not send !commands to AI fallback.
- Do not let assistant execute commands.
- Do not let assistant guess admin/economy actions.

If a known command was sent in whisper, reply in whisper if that command normally supports whisper responses.
If a known command was sent in public, keep normal command behavior.

Do not change existing command reply behavior unless needed for assistant fallback.
