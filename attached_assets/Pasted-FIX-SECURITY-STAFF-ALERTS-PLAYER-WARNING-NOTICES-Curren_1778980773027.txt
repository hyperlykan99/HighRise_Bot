PART 7 — Player Notice Delivery Rules
==================================================

- Player notice must come from ChillTopiaBot if sent by DM.
- If DM is unavailable, use private room whisper to the affected player if possible.
- Do not require the player to be subscribed to alerts.
- This is a moderation notice, not a promo/event/game notification.
- Do not send to unsubscribed broadcast users as an alert.
- Do not send duplicate notices.
- Do not crash if DM fails.
- Do not reveal staff-only internal notes unless there is a separate public reason.
- If current bot is not ChillTopiaBot/host, queue the target notice for host.
- If the target is offline and no DM conversation_id exists, log skipped safely.

Logs:

[PLAYER MOD NOTICE] action=warn target=@user delivery=dm
[PLAYER MOD NOTICE] action=warn target=@user delivery=whisper
[PLAYER MOD NOTICE SKIP] action=warn target=@user reason=no_dm_no_room_target
[PLAYER MOD NOTICE ERROR] action=warn target=@user error=...
