PART 1 — FIX MINING FORCE DROP
==================================================

Current issue:
/forcedrop is not working.

Goal:
Owner can secretly force a player’s next mine to produce a specific rarity or exact ore.

Commands to fix/keep:
- /forcedrop <username> <rarity>
- /forcedropore <username> <ore_name>
- /forcedropstatus
- /clearforcedrop <username>

Examples:
/forcedrop @player exotic
/forcedrop player prismatic
/forcedropore @player Blood Diamond
/clearforcedrop @player

Expected behavior:
/forcedrop @player exotic
→ player’s next /mine gives a random Exotic ore.

/forcedropore @player Blood Diamond
→ player’s next /mine gives Blood Diamond.

Rules:
1. Owner only.
2. Secret from players.
3. Works with manual /mine.
4. Works with AutoMine.
5. One use only.
6. Clears after successful forced mine.
7. Does not expose “forced drop” in public message.
8. Mining message should look natural.
9. Big Find announcement should trigger naturally if rarity qualifies.
10. Weight rolls normally using current ore weight system.
11. Value calculates normally.
12. FXP/MXP/rewards apply normally unless owner chooses otherwise later.
13. If player is not found, show helpful error.
14. If rarity is invalid, show valid rarities.
15. If ore name is invalid, show close suggestions.

Valid rarities:
common
rare
epic
legendary
mythic

PART 3 — MINING FORCE DROP LOOKUP
==================================================

When player mines manually or through AutoMine:

1. Check for pending forced_mining_drops by user_id first.
2. If no user_id match, check normalized username.
3. If pending forced drop exists and not expired:
   - If forced_type = rarity, pick random ore from that rarity.
   - If forced_type = ore, use exact ore.
4. Continue normal mine flow:
   - roll weight
   - calculate value
   - add inventory/record
   - add MXP
   - trigger announcements if needed
5. Mark forced drop as used.
6. Clear it from pending status.

If expired:
- mark expired
- ignore it
- mine normally
