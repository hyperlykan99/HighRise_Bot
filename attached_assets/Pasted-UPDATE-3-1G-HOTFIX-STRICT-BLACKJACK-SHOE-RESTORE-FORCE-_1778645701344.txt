==================================================
PART 1 — SHOE MUST NOT RESET ON BOT RESTART
==================================================

Critical rule:
If there is an active blackjack round, the shoe/deck must be restored exactly as saved.

The shoe must NOT be rebuilt on startup.
The shoe must NOT reshuffle on startup.
The shoe must NOT reset just because the bot restarted.

Persist the exact ordered remaining shoe as JSON.

Required saved fields:
- shoe_id
- shoe_cards_remaining_json
- shoe_discarded_json if used
- dealer_cards_json
- player_hands_json
- player_bets_json
- player_status_json
- round_id
- table_id
- phase
- action_deadline
- payouts_done
- resolved

After every card draw:
1. Remove card from shoe_cards_remaining.
2. Add card to player/dealer hand.
3. Save shoe_cards_remaining_json immediately.
4. Save hand state immediately.

Do not draw first and save later.

==================================================

On bot startup:

If active blackjack round exists:
- load saved shoe_cards_remaining_json
- load dealer_cards_json

- restart action timer
- re-show same table cards

Do NOT call create_new_shoe().
Do NOT call shuffle().

Only create new shoe if:

- refund all active bets once

==================================================

Add owner-only force card commands.

Commands:

Shows current force-card queue.

Clears all forced cards.

Forces the next drawn card.

Forces a player starting hand next round.

Example:
!bjforce player @4ktreyMarion J♦️ K♣️

!bjforce blackjack @user
Forces natural blackjack for user next round.

!bjforce suitedbj @user
Forces suited natural blackjack for user next round.

!bjforce twentyone @user
Forces regular 3-card 21 test if possible.

==================================================

- exactly 2 cards
- Ace + 10-value card
