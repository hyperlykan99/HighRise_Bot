Notification Alignment Fix — Subscription Must Be DM-Only, No Room Subscribe, Remove Inbox Summary Conflict

Current live issue:
Notification/subscription behavior is still not aligned.

Screenshots show:
1. In room chat:
!unsub changes alerts OFF.
!sub changes alerts ON.
This should NOT happen anymore.

2. In direct PM:
!sub replies:
✅ Inbox summaries enabled.
I'll DM your auto-mining and auto-fishing summaries here.

This is wrong for notification subscription.
DM !sub should control the official notification system, not a separate inbox summary system.

Goal:
Make one clean notification/subscription system.

Final rule:
Subscription management is DM-only because DM saves conversation history.

Room chat:
- !sub / !subscribe should NOT subscribe.
- !unsub / !unsubscribe should NOT unsubscribe.
- Room should only whisper instruction:
  📩 Please DM me !sub to subscribe to alerts.
  Manage alerts in DM with !notifysettings.

Direct PM:
- !sub / !subscribe turns alerts ON.
- !unsub / !unsubscribe turns alerts OFF.
- !notifysettings shows settings.
- !notify category on/off edits settings.
- random DMs are ignored.

Do NOT modify:
- poker gameplay
- blackjack gameplay
- jail gameplay
- bank transfer logic except notification side effects
- badge/title/economy logic except notification routing/help if needed

Only modify:
- modules/notif_v2.py
- modules/subscribers.py
- modules/notifications.py
- main.py DM routing if needed
- modules/multi_bot.py routing if needed
- modules/command_registry.py only if notification commands need cleanup
- modules/help_cmds.py only if needed
- any inbox summary / auto summary module only if it owns !sub in DM
