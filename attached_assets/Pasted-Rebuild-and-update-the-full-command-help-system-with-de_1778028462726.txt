Rebuild and update the full command/help system with detailed descriptions, role-based visibility, and complete owner control commands.

I want every help menu to show commands with short descriptions.

Owners should have full power to give, remove, edit, promote, demote, reset, and manage coins, event coins, XP, levels, titles, badges, roles, casino, poker, bank, events, notifications, and everything else.

IMPORTANT:
- Do not remove existing features.

- Use SQLite.
- Keep every Highrise message under 250 characters.
- Split long help menus into pages.
- Add explicit command routing before the unknown-command fallback.
- Prevent silent failures.
- Every command must reply with success, usage, permission error, or clear error.
- Do not show owner-only commands to normal players.
- Staff menus should show only commands their role can access.
- Owners should be able to view all commands.

==================================================
PART 1: ROLE HIERARCHY
==================================================

Use this role hierarchy:

1. Owner
2. Admin
3. Manager
4. Moderator
5. VIP
6. Player

Role permissions:

Owner:
- Full control.
- Can add/remove owners.
- Can add/remove admins, managers, moderators.
- Can edit coins, event coins, XP, levels, titles, badges, VIP, inventory, bank blocks, casino limits, poker settings, events, reports, and system settings.
- Can use real gold commands.

Admin:
- Can manage most economy, bank, reports, moderation, events, VIP, titles/badges, and casino settings.
- Cannot add/remove owners.
- Cannot send real Highrise gold unless also owner.
- Cannot demote/remove owners.

Manager:
- Can manage games, casino, events, poker, reports, and some moderation.
- Cannot edit player balances unless explicitly allowed.
- Cannot manage owners/admins.

Moderator:
- Can manage reports, warnings, mutes, basic resets, and view limited bank/report info.
- Cannot edit economy, roles, or real gold.

VIP:
- Player with perks only.
- No staff powers.

Player:
- Normal user.

Add helper:
get_user_role(username)
has_permission(username, required_role)
is_owner(username)
is_admin_or_higher(username)
is_manager_or_higher(username)
is_mod_or_higher(username)

Use role hierarchy consistently across all commands.
