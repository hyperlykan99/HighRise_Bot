Replace First Find System with First-To-Find Race Event System

Current issue:
The current “First Find” system behaves like a permanent first-discovery reward for a rarity.
That is not what I want.

Goal:
Convert First Find into a temporary race event system.

Meaning:
Players race during an active event to be the first player(s) to mine/catch a target rarity or specific ore/fish.

Do not change bot tokens.
Do not change room ID.
Do not reset balances.
Do not wipe player data.
Do not break mining, fishing, bank, goldtip, big announcements, auto events, or command routing.

==================================================
PART 1 — NEW CONCEPT
==================================================

Rename the concept internally if needed:

First Find Race
First To Find Event
Find Race Event

This is NOT:
- permanent first discovery
- lifetime first finder
- first player ever to find an ore

This IS:
- temporary race event
- staff starts the event
- players mine/fish during the event
- first X players to get target rarity/item win
- event ends when winners are filled or time expires

==================================================
PART 2 — EVENT EXAMPLES
==================================================

Example 1:

/setfirstfind mining prismatic 1 5
/startfirstfind 30

Meaning:
For the next 30 minutes, first 1 player to mine any Prismatic ore wins 5 gold.

Example 2:

/setfirstfind fishing exotic 3 10
/startfirstfind 60

Meaning:
For the next 60 minutes, first 3 players to catch any Exotic fish win 10 gold each.

Example 3 specific item:

/setfirstfind mining ore "Prismatic Pearl" 1 5
/startfirstfind 30

Meaning:
First player to mine Prismatic Pearl during the event wins 5 gold.

Example 4 fishing item:

/setfirstfind fishing fish "Aurora Koi" 1 5
/startfirstfind 30

Meaning:
First player to catch Aurora Koi during the event wins 5 gold.

==================================================
PART 4 — COMMAND BEHAVIOR
==================================================

/setfirstfind <mining|fishing> <rarity> <winners_count> <gold_amount>

Sets the race target by rarity.

Example:
/setfirstfind mining prismatic 1 5

Reply:
BankerBot:
🏁 First Find Race Set
Type: Mining
Target: Any Prismatic
Winners: 1
Reward: 5 gold
Use /startfirstfind <minutes> to begin.

/setfirstfinditem <mining|fishing> <item_name> <winners_count> <gold_amount>

Sets the race target by exact ore/fish.

Example:
/setfirstfinditem mining Prismatic Pearl 1 5

Reply:
BankerBot:
🏁 First Find Race Set
Type: Mining
Target: Prismatic Pearl
Winners: 1
Reward: 5 gold
Use /startfirstfind <minutes> to begin.

/startfirstfind <minutes>

Starts the active race.

Example:
/startfirstfind 30

EmceeBot announces:
🏁 First Find Race Started!
Target: Mine any [PRISMATIC] ore
Reward: 5 gold
Winners: First 1 player
Time: 30 minutes
Start mining now!

/stopfirstfind

Stops the active race manually.

EmceeBot announces:
🏁 First Find Race Ended!

/firstfindstatus

Shows current race status.

Example:
BankerBot:
🏁 First Find Race
Status: ACTIVE
Type: Mining
Target: Any Prismatic
Reward: 5 gold
Winners: 0/1
Time Left: 24m

/firstfindwinners

Shows winners.

Example:
BankerBot:
🏆 First Find Race Winners
1. @Player — Mining Prismatic — 5 gold — paid_gold

/resetfirstfind

Clears current race setup and winners.
Manager+ only.

MiningBot / GreatestProspector:
- detects mining race wins

FishingBot / MasterAngler:
- detects fishing race wins

==================================================
PART 6 — WIN DETECTION
==================================================

Only check wins while race is ACTIVE.

When player mines:
If active race type = mining:
- If target is rarity, compare mined ore rarity.
- If target is item, compare ore name.
- If match, player wins if:
  - not already a winner in this race
  - winner slots are not full

When player fishes:
If active race type = fishing:
- If target is rarity, compare fish rarity.
- If target is item, compare fish name.
- If match, player wins if:
  - not already a winner in this race
  - winner slots are not full

Do not count results before event starts.
Do not count results after event ends.
Do not count old inventory.
Only new mine/fish actions during the active race count.
