PART 2: MAKE RECOVERYSTATUS SAFE
==================================================

Implement /poker recoverystatus so it always returns a valid message.

It should handle these states:

1. No poker table exists:
"Poker recovery: no table found. No action needed."

2. Table waiting, no active hand:
"Poker recovery: table waiting | no stuck hand."

3. Active normal hand:
"Poker recovery: active hand | pot 150c | no fix needed."

4. Pot exists but cards missing:
"Poker recovery: corrupt pot 150c | use /poker hardrefund."

5. Pot exists and contributions known:
"Poker recovery: unresolved pot 150c | use /poker refund."

6. Finished hand rows stuck, pot 0:
"Poker recovery: finished stuck | use /poker clearhand."

7. Severe corruption:
"Poker recovery: severe issue | use /poker closeforce."

Never return empty string.
Never throw exception if table/hand/cards/contributions are missing.
