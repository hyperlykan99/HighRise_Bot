Title V2 Final Polish — Unify Old/New Titles, Fix Shop Titles, Move Game-Specific Titles to Achievements, Clean Commands, Perks, Stats, Logs

Current issue:
The title system is split/confusing.

GitHub currently has the old title shop in modules/shop.py:
!shop titles
!buy title title_id
!equip title title_id
!myitems
!titleinfo title_id

Old shop titles include:
rookie
lucky
grinder
trivia_king
word_master
riddle_lord
casino_rat
high_roller
millionaire
elite
immortal

Problem:
trivia_king, word_master, and riddle_lord are buyable shop titles even if players never played those games.
That is confusing. Game-specific titles should be earned, not bought.

Goal:
Finalize Title V2 so there is ONE unified title system:
- old title commands still work
- new title commands work
- shop titles are general buyable titles
- game-specific titles become achievement titles
- perks/stats/boosts are visible
- owner/admin can give/set/remove titles
- title shop bought with ChillCoins works correctly
- no duplicate ownership systems
- clean help/menu/routing

Do NOT modify:
- Poker V2 gameplay
- badge market except shared badge/title display or boost helper if needed
- jail except applying title perks if safe
- Luxe Tickets except supporter title bonus if safe
- bank transfer notifications
- notification subscription system
- unrelated games

Only modify:
- modules/shop.py if title shop remains there
- OR modules/title_system.py if cleaner
- database.py only with safe CREATE TABLE IF NOT EXISTS migrations
- main.py only for routing
- help_cmds.py only for title/global help
- fishing/mining/casino/daily/game modules only for stats/perk integration
