Fix Jail Setup Commands Not Responding Even Though route=YES handler=YES

Current in-game result:
!commandtest !setjailspot shows:
route=YES
handler=YES
security_handles=true
host_handles=false

But when owner types:
!setjailspot

Nothing happens.

Reason:
The command is owned by SecurityBot/security mode, but the owner is typing while ChillTopiaMC/host bot is active. ChillTopiaMC ignores it because host_handles=false. If SecurityBot is offline or not processing, no bot replies.

Goal:
Jail setup commands should work for owner/admin even if SecurityBot is not currently handling chat.

Keep SecurityBot as the owner for jail enforcement, but allow setup commands to be handled by host/all/security mode.

Setup commands:
- !setjailspot
- !setjailguardspot
- !setsecurityidle
- !jaildebug
- !jailadmin

Required behavior:
Owner/admin can run these commands from ChillTopiaMC or SecurityBot.

Command ownership:
- Jail runtime commands remain SecurityBot-owned:
  !jail
  !bail
  !unjail
  !jailrelease
  !jailactive
  !jailstatus
  !jailtime

- Jail setup/admin commands can be handled by:
  BOT_MODE=host
  BOT_MODE=security
  BOT_MODE=all

Why:
Setup commands only save room positions/settings. They are owner/admin-only and should not depend on SecurityBot being online.

Implementation options:

Option A — preferred:
In command ownership/default routing, set setup commands to shared/admin-safe:
!setjailspot -> host,security,all
!setjailguardspot -> host,security,all
!setsecurityidle -> host,security,all
!jaildebug -> host,security,all
!jailadmin -> host,security,all

Option B:
Before command ownership returns/ignores, add owner/admin override for jail setup commands:

if cmd in ("!setjailspot", "!setjailguardspot", "!setsecurityidle", "!jaildebug", "!jailadmin"):
    if is_owner(user.username) or is_admin(user.username):
        await jail_system.handle_setup_command(bot, user, args)
        return

Do not allow normal players to use setup commands.

Save logic:
Use existing current-user position tracking.

!setjailspot:
- save current owner position as teleport spot/key "jail"
- reply: ✅ Jail spot saved.

!setjailguardspot:
- save current owner position as "jail_guard"
- reply: ✅ Jail guard spot saved.

!setsecurityidle:
- save current owner position as "security_idle"
- reply: ✅ SecurityBot idle spot saved.

If position missing:
Reply:
⚠️ I can’t read your position yet. Move one step and try again.

Also make !jaildebug show:
jail: saved/missing
jail_guard: saved/missing
security_idle: saved/missing

Important:
Do not break:
- !tele
- !goto
- !bring
- normal teleport spots
- SecurityBot jail enforcement

Add logs:
[JAIL SETUP] current_bot_mode=host command=setjailspot owner_override=true
[JAIL SETUP] position_found=true
[JAIL SETUP] saved=jail

Tests:
