Upgrade the subscriber notification system into a full notification preferences system.

Goal:
Players should automatically become subscribed and DM-connected when they:
- tip the bot
- whisper the bot
- DM the bot
- use /subscribe

Players can choose which notifications they want ON or OFF.

IMPORTANT:
- Do not remove existing features.
- Keep bank, casino, events, gold, VIP, staff roles, shop, quests, reputation, and profiles.
- Use SQLite.
- Keep every Highrise message under 250 characters.
- Respect unsubscribe preferences.
- Prevent crashes from bad inputs.

==================================================
PART 1: AUTO SUBSCRIBE + DM CONNECT
==================================================

Auto-subscribe players when:

1. Player tips the bot
- Mark subscribed = true.
- Save username and user_id.
- If conversation_id exists, set dm_available = true.
- If no conversation_id exists, set dm_available = false.
- Message:
"✅ Alerts subscribed. DM me 'subscribe' for outside-room alerts."

2. Player whispers the bot
- Mark subscribed = true.
- Save username and user_id.
- If whisper provides conversation_id, save it.
- If not, keep dm_available based on available data.
- Message:
"✅ Alerts subscribed. Use /notifysettings to choose alerts."

3. Player DMs the bot
- Save conversation_id.
- Mark subscribed = true.
- Set dm_available = true.
- Reply:
"✅ Subscribed. Outside-room alerts are ON. Use notifysettings to choose alerts."

4. Player uses /subscribe
- Mark subscribed = true.
- If DM conversation exists, dm_available = true.
- If no DM conversation exists, tell them:
"✅ Subscribed. DM me 'subscribe' for outside-room alerts."

Unsubscribe:
If player sends in DM or room:
- unsubscribe
- unsub
- stop
- stop alerts
- optout
- opt out
- /unsubscribe

Set subscribed = false.
Keep conversation_id saved.
Reply:
"✅ Unsubscribed. All bot alerts are OFF."

Do not auto-resubscribe users who manually unsubscribed unless:
- They use /subscribe again, OR
- setting auto_resubscribe_after_tip is ON.

Default:
auto_resubscribe_after_tip = OFF.

PART 2: DATABASE
==================================================

Create or update subscriber_users table:

subscriber_users:
- username TEXT PRIMARY KEY
- user_id TEXT
- conversation_id TEXT
- subscribed INTEGER DEFAULT 1
- dm_available INTEGER DEFAULT 0
- subscribed_at TEXT
- unsubscribed_at TEXT
- last_dm_at TEXT
- last_seen_at TEXT
- auto_subscribed_from_tip INTEGER DEFAULT 0
- auto_subscribed_from_dm INTEGER DEFAULT 0
- auto_subscribed_from_whisper INTEGER DEFAULT 0
- manually_unsubscribed INTEGER DEFAULT 0

Create notification_preferences table:

notification_preferences:
- username TEXT PRIMARY KEY
- bank_alerts INTEGER DEFAULT 1
- event_alerts INTEGER DEFAULT 1
- gold_alerts INTEGER DEFAULT 1
- vip_alerts INTEGER DEFAULT 1
- casino_alerts INTEGER DEFAULT 1
- quest_alerts INTEGER DEFAULT 1
- shop_alerts INTEGER DEFAULT 1
- announcement_alerts INTEGER DEFAULT 1
- staff_alerts INTEGER DEFAULT 1
- dm_alerts INTEGER DEFAULT 1
- room_whisper_alerts INTEGER DEFAULT 1

Meaning:
- bank_alerts: coin received, bank transfer, pending deposit
- event_alerts: double XP, casino hour, auto events
- gold_alerts: gold rain, gold refund, gold tip notices
- vip_alerts: VIP gold rain, VIP events
- casino_alerts: BJ/RBJ table reminders, casino limits, casino events
- quest_alerts: completed quests, weekly rewards
- shop_alerts: shop sale, new items, VIP shop
- announcement_alerts: owner/admin announcements
- staff_alerts: staff-only alerts
- dm_alerts: allow outside-room DM messages
- room_whisper_alerts: allow room whispers

Create notification_logs table:

notification_logs:
- id INTEGER PRIMARY KEY AUTOINCREMENT
- timestamp TEXT
- username TEXT
- notification_type TEXT
- channel TEXT
  Values: room, whisper, dm, pending
- message TEXT
- status TEXT
  Values: sent, pending, skipped, failed
- error_message TEXT

Create pending_notifications table if missing:

pending_notifications:
- id INTEGER PRIMARY KEY AUTOINCREMENT
- receiver_username TEXT
- notification_type TEXT
- message TEXT
- created_at TEXT
- delivered INTEGER DEFAULT 0
- delivered_at TEXT
- delivery_attempts INTEGER DEFAULT 0
- last_error TEXT

PART 5: DELIVERY LOGIC
==================================================

Create helper function:
send_notification(username, notification_type, message, prefer_dm=True)

It should:

1. Normalize username:
- remove @
- trim spaces
- case-insensitive lookup

2. Check subscriber_users:
- If subscribed = false, skip.
- If notification type preference is OFF, skip.
- If notification type is staff and user is not staff, skip.

3. Choose delivery channel:
- If dm_alerts ON and conversation_id exists and dm_available = true:
  send outside-room DM.
- Else if user is currently in room and room_whisper_alerts ON:
  send whisper/private room message.
- Else:
  save to pending_notifications.

4. Add unsubscribe footer only to DMs:
"Stop alerts: reply unsubscribe."

5. Keep messages under 250 characters.
If too long, shorten main message first.

6. Log all attempts in notification_logs.
