Redesign Poker into a persistent real-style Texas Hold’em table.

GOAL:
Make poker feel more like a real poker table.

Current system:
- Players join one poker hand.
- Hand ends.
- Table clears.

New system:
- Players sit at the poker table with a buy-in stack.
- The poker table stays active across multiple hands.
- Players keep playing until they leave, sit out, or their table stack reaches 0.
- Small blind and big blind are posted each hand.
- Dealer button rotates.
- Bot must save the current poker table if the bot crashes/restarts.
- Bot must show players’ table stacks/balances every round so everyone can track chips.

IMPORTANT:
- Do not remove existing features.
- Keep coins, XP, bank, shop, BJ, RBJ, gold, VIP, notifications, reports, events, roles, and profiles.
- Poker uses coins only.
- Use SQLite.
- Keep messages under 250 characters.
- Prevent crashes from bad inputs.
- Add explicit command routing before unknown-command fallback.
- Poker must never deduct coins without saving state.
- Poker must never lose an active table after crash/restart.

==================================================
PART 1: NEW POKER TABLE DESIGN
==================================================

Poker should become a persistent table.

Player flow:
1. Player joins table with /p <buyin>.
2. Coins are deducted from wallet.
3. Buy-in becomes their poker table stack.
4. Table starts when enough seated players are ready.
5. Hand plays.
6. At hand end:
   - winnings/losses update table stacks.
   - players with stack > 0 stay seated.
   - players with stack = 0 are busted and removed or marked busted.
   - table starts next hand after a short delay.
7. Table only closes when:
   - fewer than min players remain, or
   - staff cancels table, or
   - everyone leaves/busts.

Do NOT return unused stack after every hand.
Only return stack when player leaves the table.

Commands:
- /p <buyin>
- /poker join <buyin>
- /poker leave
- /poker sitout
- /poker sitin
- /poker table

PART 2: PERSISTENT STACKS
==================================================

Create/update poker_seated_players table:

poker_seated_players:
- username TEXT PRIMARY KEY
- user_id TEXT
- table_stack INTEGER DEFAULT 0
- buyin_total INTEGER DEFAULT 0
- status TEXT
  Values: seated, active, folded, allin, sitting_out, busted, left
- seat_number INTEGER
- joined_at TEXT
- last_action_at TEXT
- hands_played_at_table INTEGER DEFAULT 0

Rules:
- Wallet coins are deducted only when buying in.
- Table stack is used for poker hands.
- Player wins/losses affect table_stack, not wallet directly.
- When player leaves:
  - remaining table_stack is returned to wallet.
  - remove from table or mark left.
- If player table_stack reaches 0:
  - mark busted.
  - remove before next hand unless rebuy is enabled.

Messages:
"/p 1000" response:
"♠️ @user joined Poker with 1000c stack."

/poker leave:
"👋 @user left Poker. Returned 850c."

If stack is 0:
"💀 @user busted out."

==================================================
PART 4: DEALER BUTTON AND BLIND ROTATION
==================================================

Add dealer button rotation.

Store in poker_active_table:
- dealer_button_index INTEGER
- small_blind_username TEXT
- big_blind_username TEXT
- hand_number INTEGER DEFAULT 0
- next_hand_starts_at TEXT

Hand start:
1. Get seated players with table_stack > 0 and not sitting_out.
2. Rotate dealer button each hand.
3. Assign small blind to next active player after dealer.
4. Assign big blind to next active player after small blind.
5. Deduct blinds from table stacks.
6. Add blinds to pot.
7. Set current_bet = big blind.
8. Deal hole cards.
9. First action preflop starts after big blind.

For 2-player heads-up:
- Dealer is small blind.
- Other player is big blind.
- Small blind acts first preflop.
- Big blind acts first after flop.

Messages:
"♠️ Hand #12 | Dealer @A | SB @B 50c | BB @C 100c"

If player does not have enough for blind:
- If all-in is enabled, post remaining stack as all-in blind.
- If all-in disabled, mark busted/sitting out.

PART 7: SIT OUT / SIT IN
==================================================

Add:
- /poker sitout
- /sitout
- /poker sitin
- /sitin

Rules:
- Sitting out means player stays seated but skips future hands.
- If player sits out during active hand, they fold current hand.
- Sitting out player keeps table_stack at table.
- Sitting in means they rejoin next hand if they have stack > 0.

Messages:
"⏸️ @user is sitting out."
"▶️ @user will join next hand."

If all but one player sits out:
- Finish current hand if needed.
- Pause table until 2+ active players.

Add column:

Message:
"👋 @user will leave after this hand."

After hand:
"👋 @user left Poker. Returned 850c."

PART 10: CRASH / RESTART RECOVERY
==================================================

Poker must save and restore:
- seated players
- table stacks
- sitout status
- dealer button
- small blind/big blind
- current hand number
- active round_id
- hole cards
- deck
- board
- pot
- side pots
- current bet
- current player
- all-in status
- timers
- next hand timer

Save after every action:
- join
- leave
- rebuy
- sitout/sitin
- blind posting
- cards dealt
- check/call/raise/fold/allin
- phase change
- hand end
- stack update
- next hand scheduled

On bot startup/softrestart:
- If table_status = active_hand:
  - restore current hand exactly
  - announce:
"♻️ Poker restored. Table, stacks, and cards loaded."
- If table_status = between_hands:
  - restore seated players and next hand timer
  - announce:
"♻️ Poker table restored. Next hand soon."
- If state corrupted:
  - mark recovery_required
  - do not delete stacks
  - staff can use /poker recover or /poker refund

Important:
Never lose table stacks on crash.
Never silently clear seated players.
Never duplicate payouts.

- clears active table
- logs reason poker_staff_refund
