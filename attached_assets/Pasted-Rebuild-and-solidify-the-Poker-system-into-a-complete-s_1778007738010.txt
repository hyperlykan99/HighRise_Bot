Current problems:
- Poker table can get stuck after a user auto-folds.
- Active table does not finish properly after players fold/lose.
- Player buy-ins/bets are not being added to the pot correctly.
- Bot messages are confusing.
- Poker needs limit settings like BJ/RBJ.
- Poker must survive bot crash/restart/softrestart.
- Poker hands, cards, pot, deck, bets, and player turns must be saved like RBJ.
- When bot restarts, it should notify players that the previous poker table/cards were restored.

IMPORTANT:
- Do not remove existing bot features.
- Keep coins, XP, bank, shop, blackjack, RBJ, gold, VIP, notifications, reports, events, quests, reputation, staff roles, and maintenance tools.
- Use SQLite.
- Keep every Highrise message under 250 characters.

- Add explicit command routing before unknown-command fallback.

PART 4: BUY-IN, STACK, AND POT FIX
==================================================

Fix the pot/buy-in system.

Rules:
- When a player joins lobby with /poker join <buyin>, deduct buyin immediately from their coin balance.
- Save buyin in poker_active_players.
- The buyin becomes the player’s poker stack for this round.
- Pot starts at 0 by default.
- Do NOT add entire buyin to the pot unless poker_buyin_to_pot is true.
- Default poker_buyin_to_pot = false.
- Chips go into pot only when player calls/raises/bets.
- Player stack is separate from coin balance.
- At the end of the hand:
  - Winner receives pot.
  - Each player receives their uncommitted remaining stack back.
  - Folded players lose only chips they already contributed to pot.
  - Folded players should still get uncommitted stack refunded.
- This prevents players from losing full buy-in unless they actually bet it.

Example:
Player joins with 1000c:
- coin balance decreases by 1000
- poker stack = 1000
- pot = 0

Player raises 100:
- stack = 900
- current_bet = 100
- total_contributed += 100
- pot += 100

Player folds:
- contributed chips stay in pot
- remaining stack is returned at hand end

PART 5: SIMPLE BETTING RULES
==================================================

Use simple stable betting.

Betting rounds:
- preflop
- flop
- turn
- river

Actions:
- /poker check
  Allowed only if current_bet == player.current_bet.
- /poker call
  Player contributes difference between table current_bet and player.current_bet.
- /poker raise <amount>
  Amount means raise by amount above current_bet.
  Must be between min_raise and max_raise.
  Player must have enough stack.
- /poker fold
  Player folds and cannot win the pot.

No all-in for first stable version:
- If player does not have enough stack to call/raise, say:
"Not enough poker chips."
- Do not allow all-in yet.
- No side pots yet.

Round advances when:
- All active non-folded players have acted.
- All active non-folded players have matched current_bet.
- Or only one non-folded player remains.

If only one non-folded player remains:
- Immediately finish hand.
- Award pot to remaining player.
- Return uncommitted stacks.
- Clear table.
- Do not get stuck.

Auto-fold:
- If player times out and current_bet > player.current_bet, auto-fold.
- If player times out and current_bet == player.current_bet, auto-check.
- After auto-fold/check, immediately call advance_turn_or_round().
- If only one player remains, finish immediately.

PART 6: TURN FLOW
==================================================

Create one reliable function:
advance_turn_or_round()

It must:
1. Check active non-folded players.
2. If only one active player remains:
   - finish hand immediately using reason everyone_folded.
3. If all active players matched current_bet and acted:
   - advance to next phase.
4. Phase advance:
   - preflop -> flop: deal 3 community cards
   - flop -> turn: deal 1 card
   - turn -> river: deal 1 card
   - river -> showdown
5. Reset current_bet to 0 at each new street.
6. Reset each active player current_bet to 0 at each new street.
7. Reset acted = 0 for active players at each new street.
8. Start next player turn.
9. Save table state to SQLite after each step.

This function must be used after:
- check
- call
- raise
- fold
- auto-fold
- auto-check
- timeout
- player leaves before start
- forcefinish

Do not duplicate poker flow logic across commands.

- pot
- current_bet
- current_player_index
- phase
- round_id
- lobby_ends_at
- turn_ends_at

On bot startup/on_start:
- Check poker_active_table.
- If active = 1 and phase is lobby:
  - Restore lobby players and buy-ins.
  - If lobby_ends_at still has time left, restart countdown with remaining time.
  - If lobby_ends_at expired, start the hand if enough players.
  - If not enough players, refund and clear table.

- If active = 1 and phase is preflop/flop/turn/river/showdown:
  - Restore all hands, deck, board, pot, bets, stacks, statuses, turn, timers.
  - Announce:
    "♻️ Poker table restored after restart. Previous cards loaded."
  - If it is a player's turn and turn_ends_at has time left, restart turn timer with remaining time.
  - If turn_ends_at expired, auto-check or auto-fold and continue.
  - If all active players are done, advance_turn_or_round().
  - If only one active player remains, finish hand immediately.

- If active = 1 but state is corrupted:
  - Mark phase = recovery_required.
  - Do NOT erase bets.
  - Announce to staff:
    "⚠️ Poker recovery needed. Use /poker recover or /poker refund."

Notify players after recovery:
- Public room message:
"♻️ Poker table restored. Cards, pot, and bets were loaded."
- If possible, whisper each active poker player:
"♻️ Poker restored. Use /poker hand to view your cards."

Add recovery log:

PART 12: PAYOUT SAFETY
==================================================

Prevent duplicate payouts.

Rules:
- Generate unique round_id when lobby starts or hand starts.
- Save round_id in poker_active_table and poker_active_players.
- Before paying a player, check poker_round_results.
- Never pay same round_id + username twice.
- If bot crashes during payout:
  - recovery continues unpaid results only.
  - completed payouts are not repeated.
- Log every payout/refund in economy ledger.

Payout calculations:
- Winner payout = pot share + uncommitted stack return.
- Loser/folded payout = uncommitted stack return only.
- Net = payout - original buyin.
- Update poker daily limit net using final net.

==================================================

Use asyncio.create_task.
Do not use time.sleep.

Lobby timer:
- Start when first player joins.
- Save lobby_ends_at in SQLite.
- If not enough players at end, cancel and refund.
- If enough players, start hand.

Turn timer:
- Default 20 seconds.
- Save turn_ends_at in SQLite.
- If player acts, cancel timer.
- If timer fires:
  - auto-check if no call required
  - auto-fold if call required
  - then advance_turn_or_round()

Safety:
- If timer fires but table is no longer active, do nothing.
- If timer fires but it is not that player's turn, do nothing.
- If hand already ended, do nothing.
