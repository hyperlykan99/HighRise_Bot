Add Personal Assistant AI system safely now that bots are stable.

GOAL:
Add a natural-language personal assistant so players/staff do not need to type /ai.

Users should be able to say:
"Host, help me."
"Bot, what can I do here?"
"Banker, what is my balance?"
"Poker bot, show table."
"Poker bot, why is the table stuck?"
"Miner, show my ores."
"Security, report John for spam."
"Event bot, stop autogames."

IMPORTANT STABILITY RULES:
- Do not start any new background loops.
- Do not disable SAFE_BOOT.
- Do not re-enable poker loops, autogames, bot spawns, outfit auto-apply, or emote loops.
- AI must only run when a user sends a wake phrase.
- Only Host Bot should process natural-language AI.
- Other bots should ignore natural-language messages unless they are slash commands.
- Keep exact slash commands working.
- AI should call existing command handlers instead of directly editing the database.
- Dangerous actions require confirmation.
- No duplicate AI replies.
- Keep messages under 250 characters.
- Log every AI action.
- If AI fails, it must not crash the bot.

==================================================
PART 3: AI MODES
==================================================

Add AI modes:

strict:
- Default.
- AI can answer questions.
- AI can run safe read-only commands.
- Staff/dangerous commands are suggested or require confirmation.

assistant:
- AI can execute safe player commands.
- Staff commands need permission.
- Dangerous commands require confirmation.

diagnostic:
- AI focuses on debugging and summaries.

autopilot:
- Owner only.
- AI can run safe/staff actions.
- Dangerous actions still require confirmation.

Default:
assistant_mode = strict

Commands:
- /aimode
- /aimode strict
- /aimode assistant
- /aimode diagnostic
- /aimode autopilot

Messages:
"✅ AI mode set to strict."
"AI mode: strict"

==================================================
PART 4: INTENT ROUTER
==================================================

Create a natural-language intent router.

Flow:
1. Detect wake phrase.
2. Remove wake phrase.
3. Detect target module.
4. Detect intent.
5. Build existing slash command.
6. Check permission.
7. Execute safe command or ask confirmation.
8. Log the action.

Do not directly edit DB for action commands.
Use existing command handlers.
