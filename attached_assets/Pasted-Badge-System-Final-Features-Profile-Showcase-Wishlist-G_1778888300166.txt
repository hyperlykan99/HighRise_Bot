PART 5 — Badge Gift System
==================================================

Add command:

!giftbadge @user badge_id

Example:
!giftbadge @ashquinn_ crown

Rules:
- sender must own badge
- sender cannot gift to self
- receiver must exist / be resolvable
- badge must exist
- badge must be tradeable
- badge must not be equipped by sender
- badge must not be listed in market
- badge must not be locked by sender
- receiver must not already own the same badge if duplicates are not allowed
- transfer badge ownership safely
- log gift

Messages:

Sender:
🎁 Sent badge 👑 Crown to @ashquinn_.

Receiver:
🎁 @4ktreyMarion gifted you 👑 Crown.
Equip: !equipbadge crown

If equipped:
⚠️ Unequip this badge before gifting.
Use: !unequipbadge

If listed:
⚠️ Cancel the market listing before gifting.
Use: !cancelbadge listing_id

If locked:
🔒 This badge is locked. Unlock it first:
!unlockbadge crown

If receiver already owns:
@User already owns this badge.

==================================================

Add commands:

!lockbadge badge_id

!lockedbadges

Purpose:
Prevent accidental selling/gifting.

Rules:
- player must own badge
- locked badge cannot be sold
- locked badge cannot be gifted
- locked badge can still be equipped
- lock status appears in !mybadges

PART 8 — Expensive Purchase Confirmation
==================================================

Add confirmation for expensive badge purchases.

Default threshold:
100,000 coins

Commands:
!badgebuy listing_id
!confirmbuy listing_id
!cancelbuy

If listing price >= threshold:
Do not buy immediately.

Reply:
⚠️ Confirm purchase:
💎 Diamond for 1,000,000 coins.
Type: !confirmbuy 1
Cancel: !cancelbuy

Store pending confirmation:
pending_badge_buy[user.id] = {
  listing_id,
  price,
  expires_at
}

Expiration:
60 seconds.

!confirmbuy 1:
- listing must still exist
- price must still match
- buyer must still have enough coins
- then process purchase

If expired:
Purchase confirmation expired.
Use !badgebuy 1 again.

If price/listing changed:
Listing changed. Please check !badgemarket again.

!cancelbuy:
✅ Badge purchase cancelled.

Small purchases under threshold:
Process immediately as before.

Admin setting if easy:
!setbadgeconfirm 100000

Owner/admin only.
Sets threshold.

PART 10 — Starter Badge Claim
==================================================

Add command:

!claimbadge

Purpose:
New players can claim one free starter badge.

Default starter badge:
star

If star does not exist, use first available common purchasable badge.
If no starter badge exists, admin should get safe error in console, player gets:
Starter badge is not available right now.

Rules:
- claim once per user
- user cannot claim if already claimed
- grant starter badge
- log claim

Database if safe:

CREATE TABLE IF NOT EXISTS badge_claims (
    user_id TEXT PRIMARY KEY,
    username TEXT DEFAULT '',
    badge_id TEXT DEFAULT '',
    claimed_at TEXT DEFAULT (datetime('now'))
);

Message:
✅ You claimed your starter badge: ⭐ Star.
Equip: !equipbadge star

If already claimed:
You already claimed your starter badge.

Optional:
Show in !badgehelp:
Starter: !claimbadge
