Part 4.5 — Fix AI Cost Preview Setting Intent

Current issue:
When owner says:
ai set cost preview on

AI asks:
“Which cost preview do you mean — room settings, shop item price preview, or VIP price preview?”

This is too confusing.

Goal:
When the message is inside the AI assistant and says “cost preview,” it should default to AI cost preview.

Default meaning:
cost preview = AI cost preview

Supported owner commands:
- ai set cost preview on
- ai turn cost preview on
- ai enable cost preview
- ai set cost preview off
- ai turn cost preview off
- ai disable cost preview
- ai cost preview status
- ai is cost preview on
- ai set ai cost preview on
- ai set ai cost preview off

Setting name:
AI_COST_PREVIEW_REQUIRED

Behavior:
If AI_COST_PREVIEW_REQUIRED = true:
Before any paid OpenAI answer, AI should show the cost and ask for confirmation.

Example:
Player:
ai explain quantum physics like im 10

AI:
“This costs 1 🎫 Luxe Ticket. Reply confirm to continue, or cancel.”

Player:
confirm

AI:
Calls OpenAI, answers, then deducts 1 🎫 only after success.

If AI_COST_PREVIEW_REQUIRED = false:
For 1–3 🎫 questions, AI can auto-answer and include cost note:
“Cost: 1 🎫. Quantum physics explains...”

For 5 🎫 live/current questions:
Always require confirmation regardless of setting.

Permission:
Only owner can change AI_COST_PREVIEW_REQUIRED.
Admins may view it if existing permission allows.
Players cannot change it.

Confirmation:
Changing this setting requires confirmation.

Owner:
ai set cost preview on

AI:
Prepared AI setting change:
Setting: AI Cost Preview
Current: off
New: on
Effect: Paid AI answers will show cost before charging.

Reply confirm to apply, or cancel.

Owner:
confirm

AI:
AI cost preview is now ON.

Owner:
ai set cost preview off

AI:
Prepared AI setting change:
Setting: AI Cost Preview
Current: on
New: off
Effect: 1–3 🎫 questions can auto-charge after successful answer.

Reply confirm to apply, or cancel.

Storage:
Persist this setting in room settings/database:
key: ai_cost_preview_required
value: "1" or "0"

Do not store it only in memory.

Intent routing:
If request contains:
- cost preview
- ai cost preview
- question cost preview
- ticket preview
- preview cost

Then route to:
intent = ai_cost_preview_setting

Do not ask which preview type unless the user explicitly says shop preview, VIP price preview, or room settings preview.

OpenAI intent parser:
If OpenAI sees “ai set cost preview on,” it must return:

{
  "type": "command",
  "intent": "ai_cost_preview_setting",
  "command": "set_ai_cost_preview",
  "args": ["on"],
  "target_user": null,
  "target_location": null,
  "risk": "medium",
  "needs_confirmation": true,
  "confidence": 0.95,
  "reply": "I can turn AI cost preview on after confirmation."
}

Validation:
Local bot must validate:
- user is owner
- command is whitelisted
- confirmation required
- save setting only after confirm

Debug logs:
[AI COST PREVIEW] requested=on
[AI COST PREVIEW] current=off
[AI COST PREVIEW] pending_confirmation=true
[AI COST PREVIEW] saved=on

Tests:
Owner:
- ai set cost preview on
- confirm
- ai cost preview status

Expected:
AI cost preview is ON.

Player:
- ai set cost preview off

Expected:
Owner only.

When ON:
- ai explain quantum physics like im 10

Expected:
Shows “This costs 1 🎫. Reply confirm to continue.”

When OFF:
- ai explain quantum physics like im 10

Expected:
Answers directly with “Cost: 1 🎫” and deducts after success.

Live question:
- ai weather in Tokyo today

Expected:
Always confirms because live cost is 5 🎫.
