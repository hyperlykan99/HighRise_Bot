Add Casino Integrity Checker for Poker, BJ, and RBJ.

GOAL:
Create an integrity checker that verifies Poker, Blackjack, and Realistic Blackjack are working without needing to hard-test with real players or risk real coins.

It should test:
- command routing
- command ownership
- handler exists
- database tables exist
- settings exist
- limits/toggles work
- game state can be created
- fake/simulated hand can resolve
- payouts calculate correctly
- recovery commands are available
- no duplicate bot ownership
- no silent failures

IMPORTANT:
- Do not remove existing features.
- Keep multi-bot routing.
- Poker Bot owns poker checks.
- Blackjack Bot owns BJ/RBJ checks.
- Host/Admin can run global casino check.
- Use SQLite.
- Keep messages under 250 characters.
- Do not change real player balances during integrity tests.
- Do not start public games during integrity tests.
- Use test users or dry-run transactions only.
- Every check must report PASS/FAIL clearly.
