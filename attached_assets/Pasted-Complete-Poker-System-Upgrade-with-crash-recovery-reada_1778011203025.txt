PART 6: BUY-IN, STACK, AND POT SYSTEM
==================================================

Fix buy-in and pot logic.

Rules:
- When a player joins with /poker join <buyin> or /p <buyin>, deduct buyin immediately from their coin balance.
- Save player to poker_active_players immediately.
- Save active table immediately.
- The buyin becomes player's poker stack for the round.
- Pot starts at 0 by default.
- Do NOT add entire buyin to pot unless poker_buyin_to_pot is true.
- Default poker_buyin_to_pot = false.
- Chips go into pot only through call/raise/all-in.
- Player stack is separate from coin balance.
- At hand end:
  - Winner receives pot or side pot shares.
  - Each player receives uncommitted remaining stack back.
  - Folded players lose contributed chips but get uncommitted stack back.
- Never deduct buy-in without saved table state.

Example:
Player joins with 1000c:
- coin balance -1000
- poker stack = 1000
- pot = 0

Player raises 100:
- stack = 900
- current_bet = 100
- total_contributed += 100
- pot += 100

Player folds:
- contributed chips stay in pot
- remaining stack is returned at hand end

PART 10: ADVANCE TURN OR ROUND
==================================================

Create one reliable function:
advance_turn_or_round()

It must:
1. Load latest table/player state.
2. Save state before and after changes.
3. Check active non-folded players.
4. If only one active non-folded player remains:
   - finish hand immediately using reason everyone_folded.
5. Skip folded, done, and all-in players.
6. If all active non-folded/non-allin players matched current_bet and acted:
   - advance phase.
7. Phase advance:
   - preflop -> flop: deal 3 community cards
   - flop -> turn: deal 1 card
   - turn -> river: deal 1 card
   - river -> showdown
8. Reset table current_bet to 0 at new street.
9. Reset each active player's current_bet to 0 at new street.
10. Reset acted = 0 for active non-allin players at new street.
11. If all remaining players are all-in, deal remaining board and showdown.
12. Start next player turn.
13. Save to SQLite after each step.

Use this function after:
- check
- call
- raise
- fold
- allin
- auto-fold
- auto-check
- timeout
- leave before start
- forcefinish

Do not duplicate poker flow logic across commands.

- side_pots_json
- stacks
- current_bets

- timers
- round_id

On bot startup/reconnect/softrestart:
1. Check poker_active_table.
2. If active = 1 and phase is lobby:
   - restore players and buy-ins.
   - continue countdown.
   - if expired, start if enough players, else refund.
3. If active = 1 and phase is preflop/flop/turn/river/showdown:
   - restore deck, board, hole cards, pot, side pots, stacks, bets, statuses, turn, timers.
   - announce:
"♻️ Poker restored. Cards, pot, and bets loaded."
   - whisper active players:
"♻️ Poker restored. Use /ph."
   - if turn timer expired, auto-check or auto-fold.
   - if all remaining players are all-in, continue to showdown.
4. If state corrupted:
   - mark recovery_required.
   - do not erase bets.
   - tell staff:
"⚠️ Poker recovery needed. Use /poker recover or /poker refund."

- /poker state
- /poker recover
- /poker refund
- /poker forcefinish

- refunds unresolved stacks/contributions safely.
