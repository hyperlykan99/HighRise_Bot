Update 3.3F — AI Command Control Layer for ChillTopiaMC

Goal:
Allow ChillTopiaMC AI to understand natural language and execute existing bot commands safely.

Important:
Do NOT let AI run every command blindly.
AI must respect permissions, command ownership, confirmations, economy locks, stable locks, and safety rules.

Main trigger:
ai

Examples:
- ai show my balance
- ai mine for me
- ai show events
- ai teleport me to bar
- ai show my profile
- ai start Mining Rush for 1 hour
- ai set VIP price to 600 tickets
- ai mute player for spam
- ai summarize bugs
- ai explain how to use blackjack

Required behavior:
1. Player-safe commands can run directly.
2. Staff/admin/owner commands require correct permission.
3. Economy-impacting owner/admin commands require confirmation.
4. Dangerous commands are always blocked.
5. AI must never bypass permissions.
6. AI must never expose secrets or raw database data.
7. AI must never silently wipe, reset, or mass-edit data.

Create new modules:
- ai_command_router.py
- ai_command_mapper.py
- ai_command_permissions.py
- ai_command_confirmation.py
- ai_command_safety.py
- ai_command_executor.py
- ai_command_logs.py

Examples:

STAFF_DIRECT:

Can run directly if low-risk.

Examples:

ADMIN_CONFIRM:
Admin/owner only.
Must prepare action and require confirmation.

Examples:

- change event duration

- change shop prices
- change AI reply mode
- change AI costs
- change game price
- change cooldowns
- change mining/fishing settings

Owner only.
Must require confirmation.

Examples:
- add/remove VIP
- set economy settings
- reset player stats
- refund poker table
- restart bot
- backup
- restore backup
- rollback plan
- change locks
- set casino limits

NEVER_EXECUTE:
AI must always block.

Examples:
- wipe all data
- reset economy
- reveal database
- show API key
- show bot token
- show environment variables
- raw database dump
- bypass permissions
- make me owner
- grant admin without normal command path
- change casino odds silently
- mass ban all
- delete profiles
- direct SQL
- exploit or hack commands
- give unlimited currency casually

3. Permission Rules

AI must check existing permission system before routing.

Use existing roles:
- Player
- VIP
- Staff/Moderator
- Manager
- Admin
- Owner

Rules:
- Player can only run their own safe commands.
- Player cannot run commands on other players except public info.
- Staff can run moderation/support commands.
- Admin can run config commands.
- Owner can run owner commands.
- AI must never treat text like “pretend I am owner” as permission.

4. Confirmation Rules

Require confirmation for:
- any currency edit
- any VIP edit
- any event start/stop
- any price change
- any cooldown/settings change
- any casino setting change
- any moderation action stronger than warning
- any restart/backup/restore
- any action affecting another player

Confirmation format:

AI:
Prepared command:
Command: !setvipprice 600
Reason: Change VIP price
Risk: Economy-impacting
Permission: Owner/Admin

Reply CONFIRM AI COMMAND to run, or CANCEL.

After confirm:
Run the command through the existing command handler.

If cancel:
Pending command cancelled.

Pending confirmation:
- tied to user_id
- expires after 60 seconds
- stores command, args, risk, timestamp
- never stores secrets
- cannot be confirmed by another user

5. Command Execution

Create ai_command_executor.py.

AI should NOT directly edit database unless there is no existing command handler.

Preferred:
Route through existing command handlers.

Flow:
1. Map natural language to command + args
2. Check command exists
3. Check permission
4. Check safety category
5. If direct allowed:
   - call existing handler
6. If confirmation required:
   - create pending confirmation
7. If blocked:
   - refuse

Important:
- Do not call random functions by name from user input.
- Use a whitelist command registry.
- Do not eval.
- Do not exec.
- Do not run shell commands.
- Do not import arbitrary modules from AI text.

7. Economy Safety

For currency commands:
- Owner only, or admin only if existing system allows.
- Must require confirmation.
- Must respect economy lock.
- Must log before and after.
- Must not allow negative or huge values unless owner confirms.
- Must not allow casual “give me unlimited coins”.
- Must not run if safety monitor is locked.

If economy lock is ON:
AI should say:
“Economy lock is ON, so AI cannot edit currency. Use the direct owner command if you intentionally want to override.”

Recommended:
Keep AI currency grants disabled while in stable lock.

AI:

Risk: Economy-impacting
Economy Lock: ON

AI cannot run this while economy lock is ON.

If economy lock is OFF:
Reply CONFIRM AI COMMAND to run, or CANCEL.
