==================================================

Current issue:

/bj forcedealer <card1> <card2>
/bj forcedealerace
/bj forceplayerpair
/bj forceblackjack

Goal:
Make all of them work as one-use test/debug commands for the next BlackJack deal.

Rules:
- Owner only.
- One-use only.
- Clears after next deal.
- Not shown in public help.
- Used only for testing insurance, split, bonus, blackjack payout, shoe reshuffle, and recovery.
- Must not affect normal players unless targeted.
- Must respond with confirmation when set.
- Must show helpful usage if arguments are missing.
- Must work with the next BlackJack round/deal.
- Must not crash command routing.

==================================================
1A — FORCE PLAYER CARDS
==================================================

Command:

/bj forcecards <username> <card1> <card2>

Example:

/bj forcecards @4ktreyMarion A♠ K♠

Behavior:
- Next BlackJack deal gives that target player those first two cards.
- Other players are dealt normally.
- Clears after next deal.
- If player is not in the table yet, store by normalized username and apply if they join before next deal.
- If player does not join before next deal, keep pending until one round expires, then clear safely.

Valid card formats:
A♠
K♠
Q♥
10♦
2♣

Also accept plain suits if easier:
AS
KS
QH
10D
2C

Normalize to display format.

If invalid card:

BlackJackBot:
🧪 Force Cards
Invalid card format.
Example: /bj forcecards @Player A♠ K♠

==================================================
1B — FORCE DEALER CARDS
==================================================

Command:

/bj forcedealer <card1> <card2>

Example:

/bj forcedealer A♣ K♦

Behavior:
- Next BlackJack deal gives dealer those first two cards.
- Dealer first card is visible.
- Dealer second card is hole card.
- Clears after next deal.
- Used for testing dealer BlackJack and insurance.

Confirmation:

BlackJackBot:
🧪 Dealer Cards Forced
Next dealer hand: A♣ [hidden]

==================================================

- Used for testing /insurance.
- Clears after next deal.

Confirmation:

Next round dealer will show an Ace.

PART 4 — PUBLIC CARD MODE COMPACT DISPLAY
==================================================

Current issue:
When public card mode is ON and table is full, player cards take too much space.

Goal:
In public card mode, player cards should be compact so players can easily find their own cards.

When public mode is ON:

/bj cards public

Use this compact format:

Message 1:

BlackJackBot:
🃏 Dealer Cards
Dealer: A♠ [?] | Visible Total: 11

Message 2:

BlackJackBot:
🟢 Player Cards
@Player1: 10♥ 8♣ = 18 | 🃏 /hit 🛑 /stand 💰 /double
@Player2: A♠ 7♦ = 18 | 🃏 /hit 🛑 /stand
@Player3: 9♣ 9♦ = 18 | 🃏 /hit 🛑 /stand ✂️ /split

Rules:
- Each player should use one short line when possible.
- Actions should be inline beside that player.
- Show only actions available to that player.
- Do not use a big block per player in public mode.
- Keep the last 2 BlackJackBot messages as:
  1. Dealer Cards
  2. Player Cards + Actions
- If message is too long for Highrise, split safely but avoid losing readability.
- For large tables, show compact pages if needed.

If table is too full for one message, use pages like:

🟢 Player Cards 1/2
@Player1: ...
@Player2: ...
@Player3: ...

🟢 Player Cards 2/2
@Player4: ...
@Player5: ...

But avoid sending too many messages during active decision. Prefer compact abbreviations.
