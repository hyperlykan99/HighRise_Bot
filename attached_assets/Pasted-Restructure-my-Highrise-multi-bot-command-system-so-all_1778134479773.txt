PART 1 — Create a central command registry

Create a new file if needed:
modules/command_registry.py

Each command should be registered like:
- command
- aliases
- owner_mode
- category
- handler import path or handler function name
- permission level
- fallback_allowed
- safe_help_command
- write_command

Example:
bal:
  owner_mode=banker
  handler=handle_balance or correct bank handler
  fallback_allowed=false
  write_command=false

send:
  owner_mode=banker
  handler=handle_send
  fallback_allowed=false
  write_command=true

mine:
  owner_mode=miner
  handler=handle_mine
  fallback_allowed=false
  write_command=true

pokerhelp:
  owner_mode=poker
  handler=handle_pokerhelp
  fallback_allowed=false
  safe_help_command=true

eventhelp:
  owner_mode=eventhost
  handler=handle_eventhelp
  fallback_allowed=true only if eventhost offline
  safe_help_command=true

help:
  owner_mode=host
  handler=main help handler
  fallback_allowed=true
  safe_help_command=true

PART 3 — Make one router use the registry

In main.py, replace scattered command routing with one reliable dispatch flow:

1. Parse slash command.
2. Normalize command to lowercase without slash.
3. Look up command in command_registry.
4. If unknown:
   - Host replies: Unknown command. Try /help
   - Other bots stay silent.
5. Check should_this_bot_handle(command).
6. If this bot should not handle it, return silently.
7. Check permissions.
8. Call the registered handler.
9. If handler errors, whisper a short error and log full error to console.
