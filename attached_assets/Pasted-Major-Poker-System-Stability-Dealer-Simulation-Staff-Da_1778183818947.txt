PART 3 — TURN REPEAT BUG FIX
==================================================

Problem:
After a player acts, their turn sometimes repeats.

Fix turn advancement.

Rules:
1. Each player gets only one action per turn cycle unless game rules require another action because of a new raise.
2. After action is accepted, immediately mark that player as acted for the current betting round.
3. Advance to the next eligible active player.
4. Do not prompt the same player again unless:
   - everyone else acted and a raise reopened action, or
   - it is a new betting street/round.
5. Folded players should not act.
6. All-in players should not act.
7. Players with zero chips should not act.
8. Sitting-out players should not act.

Add a per-betting-round action tracker:
- current_street: preflop/flop/turn/river/showdown
- current_actor
- last_actor
- acted_this_round set/list
- action_sequence_number
- last_action_timestamp

Add guard:
If the same action is processed twice for the same actor and same sequence number, ignore duplicate.

Log:
[POKER_TURN] actor=<username> action=<action> seq=<n>
[POKER_TURN] advancing from=<old> to=<new>
[POKER_TURN] duplicate action ignored

PART 5 — READABLE TURN PROMPTS
==================================================

Poker messages must be short because Highrise bot chat only shows small message boxes.

Turn prompt format:

“Your turn 👉 Call 100”
“Board: none yet | Your: A♠ K♦”
“Actions: /call /raise <amt> /fold /allin”

Flop/Turn/River:
“Your turn 👉 Call 100”
“Board: A♣ 7♦ 2♠ | Your: K♠ K♥”
“Hand: Pair of Kings | Win est: 42%”
“Actions: /call /raise <amt> /fold /allin”

If player can check:
“Your turn 👉 You can check”
“Actions: /check /raise <amt> /fold /allin”

If player does not have enough chips to call:
“Your turn 👉 Need 1,000 to call, you have 600”
“Actions: /allin or /fold”

Do not send long walls of text.
Split into max 2-3 short whispers if needed.
Each message must stay under Highrise message limit.
