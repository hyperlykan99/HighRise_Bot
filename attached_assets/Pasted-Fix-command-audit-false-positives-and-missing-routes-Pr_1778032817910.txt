PART 2: FIX COMMANDTEST ACCURACY
==================================================

Current bug:
/commandtest /addvip says route YES, but /addvip is unknown.

Fix:
/commandtest must check the REAL command router, not only the help registry.

It should check:
- command exists in actual routing map OR command handler condition list
- command exists in help registry
- command has permission definition
- command has a non-empty handler function

Output:
"/addvip route YES | help YES | handler YES"

If handler is missing:
"/addvip route NO | help YES | handler NO"

Do not report route YES unless typing the actual command would be caught by router.

Add debug log:
"COMMANDTEST: command=/addvip route=<bool> help=<bool> handler=<bool>"
