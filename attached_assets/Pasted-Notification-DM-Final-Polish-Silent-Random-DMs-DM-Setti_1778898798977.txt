Notification DM Final Polish — Silent Random DMs, DM Settings Editing, One Manual Subscribe System

Current live behavior:
1. In direct PM, !sub and !subscribe correctly turn alerts ON.
2. But after subscribing, random DM messages like "." can still trigger subscription/status replies.
3. Users should be able to edit notification options directly in PM.
4. Random DMs must be ignored completely.
5. There should still be only one official notification system: modules/notif_v2.py.

Goal:
Make notification DMs behave like a clean command-only system:
- Only valid notification commands trigger replies.
- Random DM messages are ignored.
- Users can subscribe/unsubscribe in DM.
- Users can view/edit notification settings in DM.
- Room notification commands still work.
- No auto-subscribe from tips, bank transfers, games, events, VIP, badges, or titles.
- No duplicate replies from old subscriber/notification modules.

Do NOT modify:
- Poker gameplay
- blackjack gameplay
- jail gameplay
- bank transfer logic except notification subscription side effects
- badge/title/economy logic except notification routing/help if needed

Only modify:
- modules/notif_v2.py
- modules/subscribers.py
- modules/notifications.py
- main.py DM routing if needed
- modules/multi_bot.py routing if needed
- modules/command_registry.py only if notification commands need route cleanup
- modules/cmd_audit.py only if notification command audit needs route cleanup
- help text only if needed

PART 12 — DM Parser Rules
==================================================

Implement a strict DM parser.

Suggested logic:

msg = message.strip()
lower = msg.lower()

If lower in {"!sub", "sub", "!subscribe", "subscribe"}:
    handle_subscribe_dm()

elif lower in {"!unsub", "unsub", "!unsubscribe", "unsubscribe", "stop", "stop alerts", "opt out"}:
    handle_unsubscribe_dm()

elif lower in {"!notifysettings", "notifysettings", "!alerts", "alerts", "settings"}:
    handle_settings_dm()

elif lower starts with "!notify " or lower starts with "notify ":
    parse category/value and handle category edit

else:
    return silently

Do not send fallback message for unknown DM.
