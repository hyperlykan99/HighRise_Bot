==================================================
PART 1 — ORE LIST DISPLAY FORMAT
==================================================

Current goal:
Make /orelist cleaner and easier to read.

New /orelist rule:
- Only the rarity header should be colored.
- Ore names should NOT be colored in /orelist.
- Color code must stop after the rarity label.
- All rarity labels must be CAPSLOCKED.
- Ore names should be one line each.
- Each ore line should show only:
  emoji/icon + ore name + chance

Do NOT show:
- ore name color
- price
- value
- weight range

Those belong in /oreprices or /oreinfo.

Rarity labels must be:

[COMMON]
[RARE]
[EPIC]
[LEGENDARY]
[MYTHIC]
[PRISMATIC]
[EXOTIC]

Color format:

Common:
<#AAAAAA>[COMMON]<#FFFFFF>

Rare:
<#3399FF>[RARE]<#FFFFFF>

Epic:
<#B266FF>[EPIC]<#FFFFFF>

Legendary:
<#FFD700>[LEGENDARY]<#FFFFFF>

Mythic:
<#FF66CC>[MYTHIC]<#FFFFFF>

Exotic:
<#FF0000>[EXOTIC]<#FFFFFF>

Prismatic:
Use this capslocked rainbow rarity label:
<#FF0000>[P<#FF9900>R<#FFFF00>I<#00FF00>S<#00CCFF>M<#3366FF>A<#9933FF>T<#FF66CC>I<#FF0000>C]<#FFFFFF>

Important:
After the rarity label, color must reset with <#FFFFFF>.
Ore names in /orelist must be plain text.

Example /orelist common:

GreatestProspector:
<#AAAAAA>[COMMON]<#FFFFFF>
🪨 Stone — 1 in 18
🟫 Clay — 1 in 18
⛏️ Coal — 1 in 18
🏜️ Sandstone — 1 in 18
💎 Quartz — 1 in 18

Requirements:
1. One ore per line.
2. Max 5 ores per page.
3. /orelist <rarity> <page> must paginate correctly.
4. Page 2 must show different ores from page 1.
5. No raw color tags should appear broken.
6. Never cut a color tag in half.
7. Always reset with <#FFFFFF>.
8. /orelist all must also be paginated.
9. /orelist menu should stay short.

PART 6 — SECRET OWNER FORCED DROP FEATURE
==================================================

Add an owner-only hidden force-next-mine system.

Purpose:
Owner can secretly make a specific player mine a specific rarity or exact ore on their next /mine.

Commands:
- /forcedrop <username> <rarity>
- /forcedropore <username> <ore_name>
- /forcedropstatus
- /clearforcedrop <username>

Examples:
/forcedrop testuser legendary
/forcedrop testuser prismatic
/forcedropore testuser Gold Ore
/clearforcedrop testuser

Behavior:
1. Owner only.
2. Secret from normal players.
3. Applies only to the player’s next /mine.
4. Clears automatically after one use.
5. Can force a random ore from a rarity.
6. Can force one exact ore.
7. Weight still rolls normally using existing weight system.
8. Value still calculates normally.
9. Mining message and announcement appear natural.
10. Player should not see “forced drop.”
11. Public chat should not reveal that the drop was forced.
12. Prevent duplicate forced drops unless owner overwrites intentionally.
13. Forced drop expires after 24 hours by default.

Example:
Owner uses:
/forcedrop testuser exotic

Then testuser uses:
/mine

They naturally mine a random Exotic ore.
The message should look like a normal Exotic mining result.

Owner status:
/forcedropstatus

Shows pending forced drops only to owner:
@testuser → rarity=exotic | expires in 23h
@player2 → ore=Gold Ore | expires in 2h

Clear:
/clearforcedrop testuser

Removes pending forced drop.

forced_type:

status:

Security:
