Notification System Final Merge — One Perfect Notify System, DM + Room Settings

Current problem:
The bot has old and new notification systems fighting each other.

Current modules involved:
- modules/notif_v2.py = new category notification system
- modules/subscribers.py = old subscribe/DM system
- modules/notifications.py = old notification helper/preferences system
- possible inbox summary / auto summary logic also uses !sub

Goal:
Merge old and new notify systems into ONE clean official system.

Official system:
modules/notif_v2.py

Final design:
- There is only one notification subscription state.
- Players can manage notifications from BOTH direct DM and room chat.
- DM replies should be sent in DM.
- Room replies should be whispered.
- Random DMs must be ignored.
- No auto-subscribe from tips, bank transfers, games, events, VIP, badges, or titles.
- No duplicate replies.
- Old notification modules must delegate to notif_v2 or become helpers only.

Do NOT modify:
- poker gameplay
- blackjack gameplay
- jail gameplay
- bank transfer logic except notification side effects
- badge/title/economy logic except notification routing/help

Only modify:
- modules/notif_v2.py
- modules/subscribers.py
- modules/notifications.py
- main.py routing if needed
- modules/multi_bot.py routing if needed
- modules/command_registry.py
- modules/cmd_audit.py
- modules/help_cmds.py
- any inbox summary module only if it conflicts with !sub / !unsub

PART 10 — Transactional Messages Stay Separate
==================================================

Bank/chillcoin transfer notices are transactional.

They may notify recipient if supported, but:
- do not require subscription
- do not subscribe user
- do not change notification settings
- do not say Alerts ON
- do not say Use notifysettings

Example:

💰 You received 500 ChillCoins from @Player.
Balance: 1,250

No subscription footer.
