PART 2: /POKER PAUSE
==================================================

Command:
- /poker pause

Permissions:
- Owner/admin/manager only.

Behavior:
- Safely pauses the poker table.
- Prevents new hands from starting.
- Pauses turn timers if possible.
- Does not delete seated players.
- Does not clear current hand.
- Does not refund automatically.
- If hand is active:
  - pause after current action if needed
  - or pause immediately but preserve turn state
- Set poker_paused = true.

Message:
"⏸️ Poker paused. Current table saved."

If already paused:
"Poker is already paused."

While paused:
- Players can use /pt, /ph, /stack.
- Players cannot /p join unless tablelock is off and staff allows.
- Betting actions should say:
"Poker is paused. Wait for staff to resume."

==================================================
PART 3: /POKER RESUME
==================================================

Command:
- /poker resume

Permissions:
- Owner/admin/manager only.

Behavior:
- Unpauses table.
- Restores timers if hand is active.
- Allows next hand to start if table is waiting and enough players are seated.
- Set poker_paused = false.

Message:
"▶️ Poker resumed."

If not paused:
"Poker is not paused."

Safety:
- Before resume, validate table state.
- If corrupted:
"Poker needs recovery. Use /poker recoverystatus."

PART 4: /POKER TABLELOCK ON/OFF
==================================================

Commands:
- /poker tablelock on
- /poker tablelock off
- /poker tablelock

Permissions:
- Owner/admin/manager only.

Behavior:
- tablelock ON prevents new players from joining.
- Existing seated players can keep playing unless poker is paused.
- Waitlist can still be allowed if enabled.
- Useful during recovery/debug.

Messages:
"🔒 Poker table locked. New joins disabled."
"🔓 Poker table unlocked. New joins enabled."

If player tries /p buyin while locked:
"Poker table is locked. Try /poker waitlist."

PART 5: /POKER WAITLIST
==================================================

Commands:
- /poker waitlist
- /poker waitlist <buyin>
- /waitpoker <buyin>
- /leavewaitlist

Player behavior:
- If table is full or locked, players can join waitlist.
- If they type /p buyin while full:
  - offer waitlist:
  "Table full. Join waitlist: /poker waitlist 1000"

- /poker waitlist <buyin>:
  - Adds player to waitlist.
  - Does not remove coins yet.
  - Saves requested_buyin.
  - Message:
  "✅ Added to poker waitlist. Position #2."

- /poker waitlist:
  - Shows waitlist:
  "Waitlist: 1 @A 1K | 2 @B 5K"

- /leavewaitlist:
  - Cancels user's waitlist entry.
  - Message:
  "✅ Removed from poker waitlist."

Staff behavior:
- When a seat opens, bot should notify next waitlisted player:
  "@user, poker seat open. Join: /p <buyin>"
- Optional auto-seat if setting enabled later, but default manual join.

Settings:
- poker_waitlist_enabled = true
- poker_waitlist_auto_notify = true
- poker_waitlist_expire_minutes = 30

Do not auto-charge coins until player actually joins.

PART 6: /POKER SPECTATE
==================================================

Commands:
- /poker spectate
- /poker spectate off
- /spectatepoker
- /spectators

Behavior:
- Lets players watch table status without joining.
- Spectators do not receive private cards.
- Spectators can see:
  - pot
  - board cards
  - players seated
  - stacks
  - current turn
  - public actions
- Spectators cannot act, bet, or see hole cards.

Messages:
/poker spectate:
"👀 You are spectating poker. Use /pt for table."

/poker spectate off:
"✅ You stopped spectating."

/spectators:
"👀 Spectators: @A, @B"

If spectate disabled:
"Poker spectate is OFF."

Privacy:
- Do not send /ph to spectators.
- Do not show private cards in spectator messages.
- Showdown only reveals cards according to poker rules.
