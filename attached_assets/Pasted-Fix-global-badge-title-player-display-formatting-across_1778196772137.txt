Fix global badge/title player display formatting across ALL systems.

Current problem:
Badge/title display is not showing in Poker and Blackjack messages.

Goal:
Whenever any bot announces, displays, calls, whispers, or mentions a player’s name, it should include that player’s equipped badge and equipped title if display toggles are ON.

Example:
Old:
@Marion called 500.
@Marion won with Blackjack.
@Marion joined poker.

New:
💎 [High Roller] @Marion called 500.
💎 [High Roller] @Marion won with Blackjack.
💎 [High Roller] @Marion joined poker.

This should happen everywhere the bot DISPLAYs a player name.
Do not change internal command parsing.
Do not change actual usernames.
Do not change balances, game rules, cards, bets, payouts, mining rewards, shop prices, event rewards, bot tokens, or room ID.

==================================================
PART 1 — VERIFY OR CREATE GLOBAL DISPLAY HELPER
==================================================

Find the existing badge/title display helper if already created.

If missing, create one shared helper, for example:

format_player_display(username, user_id=None, include_badge=True, include_title=True)

or:

get_player_display_name(username)

Recommended display format:
<badge_emoji> [<title>] @username

Examples:
💎 [High Roller] @Marion
🌟 [VIP] @Claire
⛏️ [Master Miner] @testuser
@newplayer

Rules:
1. If /displaybadges is ON and player has equipped badge, show badge emoji/icon.
2. If /displaytitles is ON and player has equipped title, show title in brackets.
3. If both OFF, show normal @username.
4. If player has no badge/title, show normal @username.
5. Do not crash if profile/badge/title data is missing.
6. Do not expose raw badge IDs unless no display name exists.
7. Keep output short.
8. If formatted name becomes too long, fallback to:
   <badge> @username
9. If still too long, fallback to:
   @username

Important:
Only use formatted names in DISPLAY messages.
Do NOT use formatted names as command arguments or database usernames.
