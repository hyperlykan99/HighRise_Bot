UPDATE 3.1G HOTFIX — UNIFY !SHOE AND !BJSHOE TO ONE BLACKJACK SHOE SOURCE

Current issue:
!shoe and !bjshoe show different blackjack shoe counts.

Live example:
!shoe shows:
Cards left: 295/312

!bjshoe shows:
Cards Remaining: 312

This proves there are two blackjack shoe sources or two shoe command handlers.

Goal:
Make !shoe and !bjshoe read the SAME blackjack shoe state.

Do not change tokens.
Do not wipe data.
Do not reset balances.
Do not change blackjack odds.
Do not reset the shoe.
Do not create a new shoe unless no saved shoe exists.

==================================================
PART 1 — SINGLE SOURCE OF TRUTH
==================================================

There must be only ONE active blackjack shoe source for AceSinatra blackjack.

Use the same source for:

!shoe
!bjshoe
!bjadmin shoe
blackjack dealing
blackjack restart restore

Preferred source:
The active realistic blackjack shoe used by AceSinatra.

Both commands must read:
- same shoe cards remaining
- same total cards
- same decks
- same used percent
- same saved/restored flags

==================================================

Make:

!shoe

Do not let !bjshoe use a new/reset DB state.

PART 5 — FIX SAVED/RESTORED FLAGS
==================================================

The display should not say confusing things.

If shoe is saved in DB:
Saved: YES

If the current in-memory shoe came from DB after startup:

If bot has not restarted since saving:
Loaded From Restart: NO is acceptable, but the card count must still match.

Important:
Saved: YES with Cards Remaining: 312 while !shoe says 295 means the wrong saved record is being read.

Fix that.

On bot startup:

2. Replace in-memory shoe with saved shoe.
3. Mark restored_from_db = true.
4. Make !shoe, !bjshoe, and !bjadmin shoe all read that same object.

Do not create separate saved shoe states.
