Update the bot help/menu system with role-based command menus.

Goal:
Players should see normal commands.
Moderators, managers, admins, and owners should see the commands they are allowed to use.

Important:
- Keep every Highrise message under 250 characters.
- Do not crash from bad inputs.
- Do not show high-level admin commands to normal players.
- Use existing permission functions if available:
  is_owner(username)
  is_admin(username)
  is_manager(username)
  is_moderator(username)
  can_manage_games(username)
  can_manage_economy(username)

If moderator role does not exist yet, add it.

Roles:
1. Owner
2. Admin
3. Manager
4. Moderator
5. Player

Permission logic:
- Owner can do everything.
- Admin can manage economy, staff, casino, games, bank, and announcements.
- Manager can manage games, casino, bank viewing, and announcements, but cannot edit coins.
- Moderator can reset games, make announcements, view limited bank/watch info, but cannot edit coins or staff roles.
- Player can only use normal player commands.
