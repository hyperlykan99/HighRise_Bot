PART 3: MULTI-BOT RUNNER
==================================================

Create a multi-bot runner.

Behavior:
1. Read all bot token env vars.
2. If only MAIN_BOT_TOKEN or original token exists:
   - start one bot in BOT_MODE=all.
3. If multiple bot tokens exist:
   - start one bot client per token.
   - assign each bot_id and bot_mode.
   - all clients share SHARED_DB_PATH.
4. Start all bots using asyncio.gather or safe async tasks.
5. If one bot fails, log error but do not crash all bots if possible.

Console examples:
"Starting Host Bot | mode host"
"Starting Blackjack Bot | mode blackjack"
"Starting Poker Bot | mode poker"

==================================================
PART 4: COMMAND HANDLING
==================================================

Every bot client must call:

should_this_bot_handle(command, bot_mode, bot_id)

Before replying.

Rules:
- BOT_MODE=all handles everything.
- BOT_MODE=blackjack handles BJ/RBJ only.
- BOT_MODE=poker handles poker only.
- BOT_MODE=miner handles mining only.
- BOT_MODE=banker handles coins/bank only.
- BOT_MODE=host handles help/profile/tutorial/welcome only.
- BOT_MODE=shopkeeper handles shop/badges/titles/VIP.
- BOT_MODE=security handles moderation/reports.
- BOT_MODE=dj handles emotes/social/dance.
- BOT_MODE=eventhost handles events/announcements.

If command is not owned by this bot:
- ignore silently.
- do not say unknown.

Only Host or all-mode bot should say:
"Unknown command. Type /help."

PART 6: SAFE DATABASE SHARING
==================================================

All bots use the same SQLite DB file.

Important:
- Use transactions for economy/game payouts.
- Use module locks for BJ, RBJ, and Poker.
- Never let two bots resolve the same game.
- If database is locked, retry briefly instead of crashing.

Use:
- bot_module_locks table
- lock module before resolving game action
- release lock after save/payout
- clear stale locks with /clearstalebotlocks
