UPDATE 3.1N — MODERATION + ANTI-ABUSE + ECONOMY SAFETY POLISH

Goal:
Protect the room economy, premium currency, player rewards, casino payouts, shops, missions, and staff tools from spam, duplicate rewards, exploit attempts, and accidental abuse.

Main features:
1. Anti-spam command protection
2. Cooldowns for spammy commands
3. Abuse detection for rewards/payouts
4. Duplicate transaction protection
5. Staff moderation tools
6. Economy exploit logs
7. Suspicious activity alerts
8. Safe admin audit commands
9. Premium currency safety
10. No reward caps

Important:
Do not change tokens.
Do not change room ID.
Do not wipe database.
Do not reset balances, Luxe Tickets, VIP, badges, titles, market, casino, blackjack, poker, mining/fishing data, collection book, missions, XP, levels, events, seasons, tools, rods, boosts, onboarding, or player inventory.
Do not re-add reward caps.
Do not add “cap reached” or “daily cap left” text.
Do not punish normal players automatically unless explicitly configured.
Do not block normal gameplay.
Do not break Luxe shop.
Do not break onboarding.
Do not break mining/fishing/casino.

Currencies:
🪙 ChillCoins = gameplay currency
🎫 Luxe Tickets = premium currency

PART 2 — COMMAND SPAM PROTECTION
==================================================

Add safe anti-spam protection for command spam.

Track per user:
- commands per 5 seconds
- commands per 30 seconds
- repeated same command count
- ignored command count
- warning count

Default thresholds:
- 5 commands in 5 seconds = soft warning
- 15 commands in 30 seconds = temporary command cooldown
- repeated same command 8 times in 30 seconds = temporary command cooldown

Response:

⚠️ Slow down. Please wait a few seconds.

Rules:
- Do not permanently ban automatically.
- Do not punish due to normal gameplay.
- Owner/Admin/Staff can be exempt or higher threshold.
- Critical commands like !help can have cooldown.
- Core action commands should not break during active games.

==================================================
PART 7 — MUTE / SOFTBAN BEHAVIOR
==================================================

Mute:
- Player can still receive basic bot responses if needed.
- Player cannot use spammy/social/public commands.
- Player cannot use !flex, !chat, assistant prompts if any.
- Player can still use safety commands like !help if allowed.

Softban:
- Player cannot earn rewards from games/mining/fishing during softban.
- Player cannot use market/trading.
- Player cannot buy/sell.
- Player can still view profile/help.
- Softban expires automatically.

Messages:
If muted:
⚠️ You are muted from public bot commands for [time].

If softbanned:
⚠️ Economy access temporarily restricted for [time].

Rules:
- Do not wipe data.
- Do not steal currency.
- Do not reset progress.
- Log action.

PART 9 — DUPLICATE TRANSACTION PROTECTION
==================================================

Prevent duplicate credits/debits for:

Verified Highrise tip events:
- event_id must be unique
- if same event_id already processed, ignore duplicate

Luxe purchases:
- unique transaction ID
- deduct once
- grant item once

Coin packs:
- deduct 🎫 once
- add 🪙 once

VIP purchases:
- deduct once
- extend VIP once

Casino payouts:
- payout once per round_id/hand_id

Mission rewards:
- claim once per mission/period

Collection milestones:
- claim once per milestone

Season rewards:
- payout once per season/category/place

Daily/streak rewards:
- claim once per day

Market/trading:
- transfer once per trade/listing purchase

Expected duplicate message:
⚠️ This reward was already processed.

Do not crash.
