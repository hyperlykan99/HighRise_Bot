Update the bot help/menu system and staff role system.

Goal:
- /help should show all player command categories.
- Staff can see every command available to their role.
- Owner/admin/manager/moderator command menus should be complete.
- Add a command to see all players with staff roles.
- Owners can add/remove another owner.

IMPORTANT:
- Do not remove existing features.
- Keep coins, XP, shop, bank, blackjack, RBJ, events, quests, reputation, reports, staff roles, and profiles.
- Use existing SQLite database.
- Use get_display_name(username).
- Keep all Highrise messages under 250 characters.
- Split long menus into pages.
- Prevent bad inputs from crashing the bot.

==================================================
PART 1: ROLE HIERARCHY
==================================================

Roles:
1. Owner
2. Admin
3. Manager
4. Moderator
5. Player

Permission rules:
- Owner can do everything.
- Admin can manage economy, bank, casino, staff below owner, events, reports, and moderation.
- Manager can manage games, casino, events, reports, announcements, and view bank/audit info.
- Moderator can announce, warn, reset games, cancel games, view reports, and view limited bank/audit info.
- Player can only use player commands.

Use or create helper functions:
- is_owner(username)
- is_admin(username)
- is_manager(username)
- is_moderator(username)
- can_manage_staff(username)
- can_manage_economy(username)
- can_manage_games(username)
- can_moderate(username)
- can_audit(username)

Suggested logic:
- can_manage_staff = owner or admin
- can_manage_economy = owner or admin
- can_manage_games = owner/admin/manager
- can_moderate = owner/admin/manager/moderator
- can_audit = owner/admin/manager/moderator
