PART 1 — Economy Safety Rules
==================================================

Poker economy must follow these rules:

1. Wallet balance:
- Deduct only when player joins or rebuys.
- Add only when player leaves, is removed, or table is closed/refunded.

2. Poker table stack:
- Winnings go to table stack first.
- Pot payout increases table stack.
- Player wallet does not receive winnings until cashout/leave.

3. Current hand contributions:
- Blinds, calls, raises, all-ins move from player table stack into pot.
- They must not be duplicated.

4. Refund:
- Current hand refund returns each player’s total_contributed back to their table stack.
- It does not credit wallet directly unless the table is closing/cashing out.

5. Cashout:
- Cashout credits wallet once.
- Then removes player from table or sets stack to 0.
- Never cash out the same stack twice.

Add safety helper:

safe_credit_wallet_once(user_id, amount, reason, ref_id)

Use a per-action ref_id to avoid duplicate credit if function runs twice.

For in-memory protection, keep:
_T["credited_refs"] = set()

If ref_id already used:
- do not credit again
- log duplicate prevented

PART 10 — Admin Close / Refund Safety
==================================================

Make sure:

!poker close:
- cancels timer/countdown
- refunds current hand contributions into table stacks
- cashes out every seated player to wallet once
- clears table
- phase = waiting
- pot = 0
- logs every cashout

!poker refund:
- cancels current hand
- returns current hand contributions to table stacks
- pot = 0
- clears board/cards/current turn
- keeps players seated
- phase = between_hands or waiting
- if 2+ players and not paused, next hand countdown starts

!poker forcefinish:
- if one eligible player remains, award pot
- if 2+ eligible and showdown valid, showdown
- if unsafe, refund hand
- log action
