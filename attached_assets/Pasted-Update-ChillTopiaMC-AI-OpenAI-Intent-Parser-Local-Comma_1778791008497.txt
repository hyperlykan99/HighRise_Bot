Flow:

3. Check host lock
4. Check safety/prompt injection locally
5. Resolve user role locally
6. Try exact local fast path
7. If fast path fails, send message to OpenAI intent parser
8. OpenAI returns strict JSON only
9. Local bot validates JSON
10. If command:
    - check whitelist
    - check permission
    - check safety

    - execute local command handler
11. If general answer:
    - show AI cost preview if needed
    - check Luxe Tickets
    - call OpenAI answer mode
    - deduct only after successful answer
12. Reply public/whisper based on reply mode

Part 3 — Teleport Command Fix

Current issue:
Using:
!tele @user [saved teleport]

does not work correctly.
It teleports me to the user instead.

Required:
Separate teleport intents clearly.

Teleport self to saved spot:
- !tele mod
- !tele bar
- !tele vip
- ai tele me to mod
- ai teleport me to bar
- ai take me to vip

Meaning:
Current user goes to saved teleport spot.

Teleport another player to saved spot:
- !tele @username mod
- !tele username mod
- ai teleport Claire to mod
- ai send Claire to vip
- ai move Claire to bar

Meaning:
Target player goes to saved teleport spot.

Go to another user:
- !goto @username
- ai go to Claire
- ai teleport me to Claire

Meaning:
Current user goes to that user.

Bring user to me:
- !bring @username
- !tphere @username
- ai bring Claire to me

Meaning:
Target user comes to current user's position.

Important parsing rule:
If command is:
!tele @username mod

Then:
target_user = username
target_location = mod
intent = teleport_other_to_saved_spot

Do NOT treat this as goto username.

If command is:
!tele @username

Then:
This is ambiguous.
Reply:
“Do you want to go to @username, or send @username to a saved teleport spot?”

If command is:
!goto @username

Then:
intent = goto_user

If command is:
!tele mod

Then:
intent = teleport_self_to_saved_spot

Permission rules:
- Normal players can teleport themselves to saved public spots.
- Normal players cannot teleport other players unless existing permission allows it.
- Staff/admin/owner can teleport other players if existing teleport permissions allow it.
- AI must not bypass teleport permissions.
