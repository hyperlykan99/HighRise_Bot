Global Economy Rescale + Mining Value Balance + Owner/Player Fairness

Focus on balancing the whole economy, especially mining values, rarity payouts, event multipliers, caps, and coin sinks.

Do not change bot tokens.
Do not change room ID.
Do not reset balances.
Do not wipe player data.
Do not remove existing player ores/items.
Do not break mining, poker, blackjack, bank, shop, events, time EXP, badges, titles, or command routing.

Goal:
Make the economy fair to players and safe for the owner.

Players should feel rewarded and excited.
Owner should be protected from inflation and unlimited coin farming.

==================================================
PART 1 — ECONOMY DESIGN GOAL
==================================================

Use this economy philosophy:

Common/Rare = steady small income
Epic = exciting good find
Legendary = strong public-announcement reward
Mythic = rare flex reward
Prismatic = jackpot tier
Exotic = ultra jackpot tier

Exotic can reach 1,000,000+ coins, but only when:
- the ore is rare/top-tier
- the weight roll is high
- or an event boost is active

Do NOT make every Exotic automatically worth 1,000,000+ coins.

==================================================

Rescale mining values so average final values after weight are roughly:

Common:
- Average final value: 10–100 coins
- Max normal value: 200 coins

Rare:
- Average final value: 150–1,000 coins
- Max normal value: 2,500 coins

Epic:
- Average final value: 2,000–10,000 coins
- Max normal value: 25,000 coins

Legendary:

- Max normal value: 250,000 coins

Mythic:
- Average final value: 100,000–350,000 coins

Prismatic:

- Max normal value: 1,500,000 coins

Exotic:
- Average final value: 750,000–2,000,000 coins
- Max normal value: 5,000,000 coins

Important:
These are target ranges after weight calculation, not necessarily base values.

==================================================
PART 3 — EXOTIC VALUE RULE
==================================================

Exotic should be the top rarity.

Recommended Exotic behavior:
- Weak Exotic + low weight = 500k–900k
- Normal Exotic = 1M–2M
- Heavy/top Exotic = 3M–5M

Example:
Blood Diamond base value: 100,000
Weight: 5.2kg
No event:
100,000 × 5.2 = 520,000 coins

With Ore Value Surge 1.5x:
520,000 × 1.5 = 780,000 coins

With Admin’s Mining Blessing 2x:
520,000 × 2 = 1,040,000 coins

Top Exotic example:
Forbidden Core base value: 250,000
Weight: 4kg
Final value:
250,000 × 4 = 1,000,000 coins

This means 1M+ is possible and exciting, but not automatic.

==================================================
PART 4 — FINAL VALUE FORMULA
==================================================

Use this formula:

final_value = base_value × weight_multiplier × event_multiplier

Recommended:
weight_multiplier = weight

If current economy becomes too strong, use configurable scale:

final_value = base_value × (1 + ((weight - 1) × weight_value_multiplier_scale))

Default:
weight_value_multiplier_scale = 1.0

Round final value to a clean integer.

Do not retroactively change value of old inventory unless the sell system already calculates value at sell time.

PART 7 — EVENT MULTIPLIER BALANCE
==================================================

Events should help players but not break the economy.

Recommended event balance:

Ore Value Surge:
- Normal auto event value boost: 1.5x
- Not 2x by default

Admin’s Mining Blessing:
- Admin/owner-controlled event
- Can use 2x ore value

Prismatic Hunt:
- 2x Prismatic chance
- Still rare

Exotic Hunt:
- 2x Exotic chance
- Still extremely rare

Lucky Rush:
- +25% mining luck

Heavy Ore Rush:
- +25% weight luck

Double MXP:
- 2x mining EXP only, not coin value

Mining Haste:
- -25% cooldown
- Must respect minimum cooldown

==================================================
PART 8 — MULTIPLIER STACKING CAPS
==================================================

Prevent infinite stacking.

Add final multiplier cap.

Normal event multiplier cap:
- max 3x total value multiplier

Admin’s Mining Blessing cap:
- max 5x total value multiplier

Example:
If player has item boost + event boost + blessing:
combined multiplier should not exceed cap.

Stacking rule:
Use additive percentage boosts when possible.

Example:
Item boost +20%
Event boost +50%

boost_multiplier = 1 + 0.20 + 0.50 = 1.70

Do not multiply endlessly like:
1.2 × 1.5 × 2 × 2 unless capped.
