Notification System Remodel — Manual Subscribe Only + Categories + Broadcasts

Current issue:
Tipping still auto-subscribes players and repeatedly DMs:

✅ Alerts ON. Use notifysettings to choose alerts. Stop: reply unsubscribe.

This must stop.

Goal:
Rebuild/clean the notification system so:
1. Players only subscribe manually.
2. Tipping never auto-subscribes anyone.
3. Notifications are direct messages/whispers.
4. Notification categories are configurable.
5. Owner/admin can send announcements, promos, events, games, and tip alerts.
6. Bank transfer notifications stay separate and always send as transactional messages.

Do NOT modify:
- Poker V2
- blackjack
- jail logic
- Luxe Ticket conversion except removing notification auto-subscribe from tip flow
- bank transfer transactional notifications except making sure they bypass subscription checks
- unrelated games

Only modify:
- notification / alert modules
- DM command handling
- help text
- tip handler only where it triggers notification auto-subscribe
- database.py if safe migration is needed
- main.py only if routing is needed

PART 6 — Change Category Settings
==================================================

Command:

!notify [category] [on/off]

Examples:
!notify events on
!notify games off
!notify announcements on
!notify promos off
!notify tips on

Rules:
- User must be subscribed first.
- Category must be valid.
- on/off must be valid.
- Save settings.

Messages:
✅ Events alerts ON.
✅ Games alerts OFF.
✅ Announcements alerts ON.
✅ Promo alerts OFF.
✅ Tip alerts ON.

Invalid category:
Use: !notify events/games/announcements/promos/tips on/off

Invalid value:
Use: !notify events on or !notify events off

PART 7 — Subscription Database
==================================================

If database.py can be safely modified, add:

CREATE TABLE IF NOT EXISTS notification_subscriptions (
    user_id TEXT PRIMARY KEY,
    username TEXT DEFAULT '',
    subscribed INTEGER DEFAULT 0,
    events INTEGER DEFAULT 1,
    games INTEGER DEFAULT 1,
    announcements INTEGER DEFAULT 1,
    promos INTEGER DEFAULT 1,
    tips INTEGER DEFAULT 0,
    source TEXT DEFAULT 'manual',
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);

Rules:
- subscribed = 1 only from manual !sub / !subscribe
- subscribed = 0 from !unsub / unsubscribe / stop
- source must be manual
- never set source = tip
- never create subscribed=1 from tip events

If DB fails:
- do not crash bot
- print warning
- safe default: not subscribed

PART 10 — Broadcast Rate Safety
==================================================

Do not blast all users at once.

When broadcasting:
- send one DM at a time
- sleep 0.3 to 0.7 seconds between sends
- catch exceptions
- continue if one DM fails

Log:
[NOTIFY BROADCAST] category=announcements delivered=12 failed=1 skipped=4

PART 14 — Bank Transfer Notifications Stay Separate
==================================================

Bank transfer notifications are transactional.

They should always be attempted:
- sender confirmation
- receiver confirmation

They do NOT require:
!sub
subscribed = 1
category ON

Do not route bank transfers through promotional notification subscription checks.

Code comment:
# Transactional bank messages bypass notification subscriptions.

Examples:
Sender:
✅ Sent 5,000 ChillCoins to @User.
Balance: 25,000 coins.

Receiver:
💰 @Sender sent you 5,000 ChillCoins.
Balance: 12,500 coins.
