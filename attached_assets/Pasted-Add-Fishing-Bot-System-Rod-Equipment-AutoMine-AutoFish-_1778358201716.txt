Add Fishing Bot System + Rod Equipment + AutoMine/AutoFish Safety Limits

Focus on adding a new Fishing Bot system similar to Mining Bot, plus safer AutoMine and AutoFish systems so players do not get muted by Highrise for repeating the same command.

Do not change bot tokens.
Do not change room ID.
Do not reset balances.
Do not wipe player data.
Do not break mining, poker, blackjack, bank, shop, events, AI, security, outfits, bot spawns, or command routing.

Main goals:
1. Add a Fishing Bot similar to Mining Bot.
2. Add fish rarities, fish weights, fish values, fish list, fish prices, fish info.
3. Add fishing EXP/level system separate from mining.
4. Add fishing rod equipment with unique rod names.
5. Add fishing leaderboards.
6. Add fishing announcements for rare catches.
7. Add AutoMine with safety limits.
8. Add AutoFish with safety limits.
9. AutoMine/AutoFish should prevent players from typing the same command repeatedly and getting warned/muted by Highrise.
10. AutoMine/AutoFish must stop when the player leaves the room.
11. AutoMine/AutoFish must be adjustable by staff.

PART 7 — FISH WEIGHT AND VALUE SYSTEM
==================================================

Every caught fish should generate a weight.

Use lb for fishing by default.

Fish value should scale with weight.

Suggested formula:
final_value = base_value × weight_multiplier × event_multiplier × rod_value_bonus

Simple option:
weight_multiplier = weight

Safer option:
final_value = base_value × (1 + ((weight - 1) × fishing_weight_value_scale))

Default:
fishing_weight_value_scale = 1.0

Every caught fish should save:

PART 16 — AUTOMINE SYSTEM
==================================================

Problem:
Highrise may warn or mute players if they type the same command 3 times fast, like:
/mine
/mine
/mine

Goal:
Add AutoMine so players type one command and the bot handles mining attempts safely.

Player commands:
- /automine on
- /automine off
- /autominestatus

Aliases:
- /am on
- /am off
- /amstatus

Behavior:
/automine on:
Starts automatic mining session.

/automine off:
Stops automatic mining session.

/autominestatus:
Shows current session status.

AutoMine must:
1. Respect mining cooldown.
2. Respect equipped mining tool cooldown.
3. Respect active mining events.
4. Respect daily mining limits if any.
5. Not bypass normal mining rules.
6. Not continue offline.
7. Stop when player leaves the room.
8. Stop when player disconnects.
9. Stop if bot cannot confirm player is present.
10. Not restart automatically after bot crash/restart.
11. Require player to type /automine on again after returning.
12. Use one active AutoMine session per player only.

Default limits:
- AutoMine enabled: ON
- Session duration: 30 minutes
- Max attempts per session: 30 mines
- Daily AutoMine cap: 120 minutes
- Stop on player leave: YES
- Offline mining: NO
- Restart after crash: NO

AutoMine should stop when the first limit is reached:
- 30 minutes OR
- 30 mining attempts OR
- daily cap reached OR
- player leaves room

Start message:

GreatestProspector:
⛏️ AutoMine Started
Limit: 30 mines or 30 minutes.
Stops if you leave the room.
Use /automine off to stop.

Status message:

GreatestProspector:
⛏️ AutoMine
Status: ON
Session: 12/30 mines
Time: 8m/30m
Daily: 35m/120m
Stops if you leave room.

Stop message:

GreatestProspector:
⛏️ AutoMine Stopped
Reason: Session limit reached.
Mines completed: 30/30

If player leaves:

GreatestProspector:
⛏️ AutoMine Stopped
Reason: Player left the room.
Mines completed: 14/30

==================================================

Prevent players from getting warned/muted by Highrise for repeating commands.

1. Encourage /automine on instead of repeated /mine.
2. Encourage /autofish on instead of repeated /fish.
3. Valid repeated game commands should not trigger bot mute.
4. Cooldown replies should be throttled.
5. Do not send a cooldown reply every time if player spams.
6. Use one cooldown reply every few seconds per user/command.
7. Do not hard mute players for valid repeated commands like:
   /mine
   /fish
   /hit
   /stand
   /double
   /split
   /bet
8. Warn/cooldown instead of mute when possible.
