    return (
        normalized == "ai"
        or normalized.startswith("ai ")

        or normalized.startswith("ai:")
        or normalized.startswith("ai?")
        or normalized.startswith("ai!")

        or normalized.startswith("assistant ")
        or normalized.startswith("bot ")

Step 5 — Knowledge access layer

Create permission-controlled knowledge levels.

Knowledge levels:

PUBLIC:
Available to everyone.

Includes:
- command explanations
- game guides
- Luxe Tickets explanation
- ChillCoins explanation
- mining guide
- fishing guide
- casino guide
- event guide
- VIP guide

PLAYER_PRIVATE:
Available only to the same player.

Includes:
- own ChillCoins balance
- own Luxe Tickets balance
- own level
- own mission progress
- own inventory summary
- own cooldowns
- own rewards status

STAFF:
Available to staff/admin/owner.

Includes:
- open reports
- bug summaries
- moderation suggestions
- recent warnings
- support queue
- player support issues

ADMIN:
Available to admin/owner.

Includes:
- event settings
- shop settings
- VIP settings
- assistant settings
- non-sensitive economy settings
- event schedule settings

OWNER:
Available only to owner.

Includes:
- economy dashboard
- analytics
- protected logs
- advanced configuration summaries
- owner-only system summaries

NEVER_EXPOSE:
Never reveal through AI chat.

Includes:
- API keys
- tokens
- passwords
- database credentials
- raw database dumps
- private environment variables
- full hidden casino formulas if not public
- other players’ private info
- exploit-sensitive anti-abuse logic
- raw JSON database records
- internal secrets
- backend file paths that expose sensitive structure

Before answering any question, AI must:

1. Detect intent
2. Detect requested knowledge type
3. Check user permission
4. Fetch only allowed context
5. Refuse or simplify if not allowed
