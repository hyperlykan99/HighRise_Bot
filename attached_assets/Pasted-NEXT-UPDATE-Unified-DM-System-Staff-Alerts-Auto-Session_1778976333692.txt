PART 1 — One DM Sender Rule
==================================================

All direct messages must come from ChillTopiaBot / host bot only.

This includes:

- !sub / !unsub DM replies
- broadcast announcements
- event/game/promo/tip alerts
- help/guide DM replies
- auto-mine summaries
- auto-fish summaries
- requested summaries
- staff alerts
- report alerts
- bot health alerts
- QA failure alerts

Other bots must NOT send direct messages.

Examples:
MinerBot may handle mining.
FisherBot may handle fishing.
PokerBot may handle poker.
BankerBot may handle economy.
But ChillTopiaBot is the only DM sender.

If another bot needs to send a DM:
- save/queue the message for ChillTopiaBot
OR
- call a shared DM queue helper that only ChillTopiaBot processes

Do not let multiple bots DM the same player.

Add logs:
[DM ROUTE] queued_for_host user=@name type=auto_fish_summary
[DM HOST SEND] user=@name type=auto_fish_summary status=sent
[DM BLOCKED] bot=fisher reason=host_only_dm
