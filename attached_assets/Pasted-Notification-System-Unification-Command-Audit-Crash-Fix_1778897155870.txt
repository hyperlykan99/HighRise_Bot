There are currently multiple overlapping notification systems:

1. modules/subscribers.py
- old subscriber DM system
- handles !subscribe, !unsubscribe, !substatus, !subhelp
- handles !announce_subs, !announce_vip, !announce_staff
- handles pending subscriber messages
- contains old auto_subscribed_from_tip compatibility fields

2. modules/notif_v2.py
- newer manual-subscribe-only notification system
- supports categories: events, games, announcements, promos, tips
- handles !sub, !unsub, !notifysettings, !alerts, !notify category on/off
- handles !announcement, !promo, !eventalert, !gamealert, !tipalert
- should become the official system

3. modules/notifications.py
- older central delivery + preference system
- has send_notification()
- has old preference fields: bank_alerts, event_alerts, gold_alerts, vip_alerts, casino_alerts, quest_alerts, shop_alerts, announcement_alerts, staff_alerts, dm_alerts, room_whisper_alerts
- has pending notifications and old notify commands
