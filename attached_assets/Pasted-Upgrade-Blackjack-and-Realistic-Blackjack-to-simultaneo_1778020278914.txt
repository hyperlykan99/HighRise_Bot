Upgrade Blackjack and Realistic Blackjack to simultaneous player action, add split, and add short safe commands.

GOAL:
Change BJ/RBJ so players do NOT wait for turns after cards are dealt.
After the round starts, every active player can act anytime during the action timer:
- hit
- stand
- double
- split

Also add short commands that are easy to type and do not interfere with poker or other games.

IMPORTANT:
- Do not remove existing features.
- Keep coins, XP, bank, shop, poker, gold, VIP, events, notifications, reports, roles, and profiles.
- Apply to both Casual Blackjack /bj and Realistic Blackjack /rbj.
- Use SQLite.
- Keep every Highrise message under 250 characters.
- Prevent crashes from bad inputs.
- Add explicit routing before unknown-command fallback.
- Maintain BJ/RBJ crash recovery and saved table state.
- Never deduct bets without recovery/refund option.

==================================================
PART 1: CHANGE BJ/RBJ FROM TURN-BASED TO SIMULTANEOUS ACTION
==================================================

Current issue:
BJ/RBJ waits for each player turn. I want everyone in the table to act anytime after cards are dealt.

New flow:
1. Players join lobby.
2. Lobby countdown ends.
3. Bot deals cards to all players and dealer.
4. Bot starts a shared action timer.
5. During the action timer, any player can:
   - hit
   - stand
   - double
   - split
6. A player can keep hitting until they stand, bust, double, split-hand resolves, or timer expires.
7. When all players are done OR timer expires:
   - dealer reveals cards
   - dealer plays
   - bot resolves all player hands
   - bot pays/refunds/losses
   - table clears

Default shared action timer:
- bj_action_timer = 30 seconds
- rbj_action_timer = 30 seconds

Settings commands:
- /setbjactiontimer <seconds>
- /setrbjactiontimer <seconds>

Validation:
- Minimum 10 seconds
- Maximum 90 seconds
- Owner/admin/manager only

Messages:
When action phase starts:
"🃏 BJ action open! Act within 30s: hit, stand, double, split."

RBJ:
"🃏 RBJ action open! Act within 30s: hit, stand, double, split."

When timer expires:
"⏰ Action time over. Dealer plays."

PART 3: HIT RULES
==================================================

Commands:
- /bj hit
- /rbj hit
- short aliases listed later

Behavior:
- Player can hit anytime during action phase if they have an active hand.
- Add one card to their current active hand.
- Save state immediately.
- If hand total > 21:
  - mark hand busted
  - move to next active split hand if any
  - if no active hands left, player done
- If hand total == 21:
  - auto-stand that hand
  - move to next active split hand if any

Messages:
"🃏 @user hits: 8♠ 7♦ 5♣ = 20"
"💥 @user busts: 24."

Keep under 250 characters.

PART 5: DOUBLE RULES
==================================================

Commands:
- /bj double
- /rbj double
- short aliases listed later

Behavior:
- Player can double only if:
  - hand has exactly 2 cards
  - hand is active
  - player has enough coins to add another bet equal to hand bet
  - double is not disabled
- Deduct extra bet.
- Add extra bet to hand bet.
- Deal exactly one card.
- Mark hand doubled/done unless bust.
- Move to next active split hand if any.
- Save state immediately.
- If all players done, resolve.

Messages:
"💰 @user doubles to 200c and draws 6♣. Total: 18."
"❌ Double allowed only on first 2 cards."
"❌ Not enough coins to double."

Settings:
- /bj double on/off
- /rbj double on/off
Owner/admin/manager only.

Defaults:
- bj_double_enabled = true
- rbj_double_enabled = true

==================================================
PART 6: ADD SPLIT FEATURE
==================================================

Commands:
- /bj split
- /rbj split
- short aliases listed later

Behavior:
Player can split if:
- current active hand has exactly 2 cards
- both cards have same rank
  Examples: 8+8, K+Q? No. Same rank only by default.
- player has enough coins to place an additional bet equal to original hand bet
- split is enabled
- player has not exceeded max splits

Default:
- max_splits_per_player = 1
- split_enabled = true
- split_aces_one_card = true

Settings:
- /bj split on/off
- /rbj split on/off
- /setbjmaxsplits <amount>
- /setrbjmaxsplits <amount>

Validation:
- max splits 0 to 3
- Owner/admin/manager only

Split rules:
1. Deduct additional bet equal to current hand bet.
2. Separate the two cards into two hands.
3. Deal one card to each split hand.
4. Player continues playing the first active split hand.
5. After first hand stands/busts/doubles, move to next split hand.
6. Save state immediately.

Split aces:
- If split_aces_one_card = true:
  - split aces receive one card each
  - both hands auto-stand
- Add setting:
  /bj splitaces on/off
  /rbj splitaces on/off
Where ON means one-card split aces rule is active.

Messages:
"✂️ @user splits 8s. New bet: 100c each."
"❌ Split needs two matching ranks."
"❌ Not enough coins to split."

PART 7: SIMULTANEOUS ROUND RESOLUTION
==================================================

Round should resolve when:
- all players have no active hands
OR
- shared action timer expires

Resolution:
1. Auto-stand remaining active hands if timer expired.
2. Dealer reveals.
3. Dealer hits according to rules.
4. Compare each player hand to dealer.
5. Pay each hand separately.
6. Blackjack, bust, win, loss, push are calculated per hand.
7. Update stats and daily limits.
8. Prevent duplicate payouts.
9. Clear table and timers.

Dealer rules:
- Dealer hits until 17.
- If existing settings for soft 17 exist, respect them.
- Otherwise stand on all 17.

Messages:
"Dealer: A♠ 6♦ K♣ = 17."
"@user H1 wins +100c. H2 push."
"@user lost 100c."
"BJ round ended."

Keep messages short. If many players/hands, send multiple short messages.

PART 9: SHORT SAFE COMMANDS
==================================================

Add shorter commands without interfering with poker.

BJ short aliases:
- /bjoin <bet> -> /bj join <bet>
- /bt -> /bj table
- /bh -> /bj hit
- /bs -> /bj stand
- /bd -> /bj double
- /bsp -> /bj split
- /blimits -> /bj limits
- /bstats -> /bj stats

RBJ short aliases:
- /rjoin <bet> -> /rbj join <bet>
- /rt -> /rbj table
- /rh -> /rbj hit
- /rs -> /rbj stand
- /rd -> /rbj double
- /rsp -> /rbj split
- /rshoe -> /rbj shoe
- /rlimits -> /rbj limits
- /rstats -> /rbj stats

Do NOT use these aliases because they conflict with poker:
- /h
- /s
- /d
- /sp
- /hit
- /stand
- /double
- /split

Reason:
Poker already uses /check, /call, /raise, /fold, /ai. Do not create generic aliases that could confuse game context.

Alias safety:
- /bh should only work if user is in an active BJ round.
- /rh should only work if user is in an active RBJ round.
- If not in that table:
  "You are not in BJ."
  or
  "You are not in RBJ."

Join alias validation:
- /bjoin 100 requires positive number.
- /rjoin 100 requires positive number.
- If missing:
  "Use /bjoin <bet>."
  "Use /rjoin <bet>."

Routing:
- Add explicit routes before unknown-command fallback.
- Short aliases must call the same BJ/RBJ functions.
- Do not duplicate game logic.
