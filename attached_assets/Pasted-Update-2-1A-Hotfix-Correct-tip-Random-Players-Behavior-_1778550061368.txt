Update 2.1A Hotfix — Correct !tip Random Players Behavior

Current issue:
!tip 5 5 should tip random players, but it is not completing the full tipping flow correctly.

Required behavior:

!tip 5 5

Means:
Tip 5 random eligible real players with 5 gold each.

Format:
!tip [number_of_players] [amount_each]

Examples:
!tip 5 5 = tip 5 random players, 5g each
!tip 3 10 = tip 3 random players, 10g each
!tip all 1 = tip all eligible real players, 1g each
!tip @user 5 = tip one player, 5g

==================================================
PART 1 — RANDOM PLAYER TIP FLOW
==================================================

When owner or active Party Tipper types:

!tip 5 5

Do this:

1. Check Party Mode is ON.
2. Check user is Owner or active Party Tipper.
3. Check Party Wallet has enough gold.
4. Build eligible room player list.
5. Exclude all bots.
6. Exclude ChillTopiaMC.
7. Exclude BankerBot/BankingBot.
8. Exclude all known bot usernames.
9. Exclude usernames ending with "bot".
10. Exclude self if self-tip is OFF.
11. Randomly select up to 5 eligible players.
12. Tip each selected player 5g.
13. After each successful tip, ChillTopiaMC publicly says:

💸 Tipped @username 5 gold!

14. After all tips, send final summary.

==================================================
PART 2 — IF NOT ENOUGH ELIGIBLE PLAYERS
==================================================

If user types:

!tip 5 5

but only 3 eligible real players are in the room:

Do not stop.

First say:

⚠️ Only 3 eligible player(s) found.
Tipping 3 player(s) instead.

Then continue tipping those 3 players.

Example:

💸 Tipped @Player1 5 gold!
💸 Tipped @Player2 5 gold!
💸 Tipped @Player3 5 gold!

Then final summary:

🎉 Party Tip Random
Players tipped: 3
Amount each: 5g
Total: 15g
Party Wallet Left: [remaining]g

Only stop if eligible count is 0.

If eligible count is 0:

⚠️ No eligible real players found.
Bots are excluded from all gold tipping.
