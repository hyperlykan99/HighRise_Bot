PART 7 — BOT MAINTENANCE / SHUTDOWN CONTROLS
==================================================

Add a feature to disable/shutdown bots for maintenance.

Goal:
Owner/Manager should be able to put one bot, one system, or all bots into maintenance mode safely without wiping data or breaking restart recovery.

Commands:

/maintenance
/maintenance status
/maintenance on
/maintenance off
/maintenance bot <botname> on
/maintenance bot <botname> off
/maintenance all on
/maintenance all off
/botstatus

Optional only if safely supported:

/shutdownbot <botname>
/startbot <botname>
/restartbot <botname>

==================================================
7A — PERMISSIONS
==================================================

- Owner only for full shutdown/restart commands.
- Manager+ can view status.
- Manager+ can enable maintenance mode if safe.
- If shutdown/start/restart cannot be safely supported, do not register those commands as active.

==================================================
7B — GLOBAL MAINTENANCE MODE
==================================================

When global maintenance is ON:
- Block non-staff gameplay commands.
- Keep staff commands working.
- Keep /help, /start, /maintenance status, /botstatus working.
- Show clear message to players.

Player message:

🛠️ Maintenance Mode
Bots are being updated. Please try again soon.

Do not block owner/manager from turning maintenance off.

==================================================

==================================================
7E — SHUTDOWN VS MAINTENANCE
==================================================

Important distinction:
- maintenance on = bot still online, commands blocked safely.
- shutdownbot = attempt to stop that bot process only if project architecture supports it.

If actual process shutdown is not safely supported, implement maintenance mode only and say:

Actual bot process shutdown is not supported safely from chat. Maintenance mode enabled instead.

Do not kill processes unless safe and supported.
